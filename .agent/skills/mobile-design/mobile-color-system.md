# 移动端色彩系统参考 (Mobile Color System Reference)

> OLED 优化、深色模式、电池感知色彩与户外可见性。
> **移动端的色彩不仅是美学——它关乎电池寿命和可用性。**

---

## 1. 移动端色彩基础

### 为什么移动端色彩不同

```
桌面端 (DESKTOP):                  移动端 (MOBILE):
├── LCD 屏幕 (背光)                ├── OLED 普遍 (自发光)
├── 受控的光照环境                 ├── 户外，强光直射
├── 稳定的电源                     ├── 电池续航至关重要
├── 个人偏好                       ├── 系统级深色模式
└── 静态观看                       └── 视角多变，运动中
```

### 移动端色彩优先级

| 优先级          | 为什么               |
| :-------------- | :------------------- |
| **1. 易读性**   | 户外，多变的光照     |
| **2. 电池效率** | OLED = 深色模式省电  |
| **3. 系统集成** | 深色/浅色模式支持    |
| **4. 语义**     | 错误, 成功, 警告颜色 |
| **5. 品牌**     | 在满足功能需求之后   |

## 2. OLED 考量

### OLED 与 LCD 的区别

```
OLED (有机发光二极管):
├── 每个像素自己发光
├── 黑色 = 像素关闭 (零耗电)
├── 耗电 = 像素越亮耗电越多
└── 深色模式 = 显著省电
```

### 电池节省与 OLED

```
色彩能耗 (相对):

#000000 (纯黑)        ████░░░░░░  0% (最省电)
#1A1A1A (近黑)        █████░░░░░  ~15%
#333333 (深灰)        ██████░░░░  ~30%
#FFFFFF (纯白)        ██████████  100%

饱和色也极其耗电:
├── 蓝色像素: 效率最高
├── 绿色像素: 中等
├── 红色像素: 效率最低
└── 去饱和颜色更省电
```

### 纯黑 (#000000) vs 近黑 (#121212)

```
#000000 (纯黑):
├── 最大化电池节省
├── 在滚动时可能导致 "黑色拖影 (Black Smear)"
├── 对比度极高 (可能刺眼)
└── Apple 的纯深色模式在使用

#121212 (近黑 - Material 推荐):
├── 依然有不错的省电效果
├── 滚动更平滑 (无拖影)
├── 对眼睛稍微柔和
└── Material Design 推荐

**推荐:** 背景使用 #000000 或 极深灰，表面 (Surface) 使用 #1C1C1E。
```

## 3. 深色模式设计

### 深色模式色彩策略

```
浅色模式 (LIGHT)                深色模式 (DARK)
──────────                      ─────────
背景:       #FFFFFF      →      #000000 或 #121212
表面:       #F5F5F5      →      #1C1C1E
表面 2:     #EEEEEE      →      #2C2C2C

主色:       #1976D2      →      #90CAF9 (变浅/减淡)
文字:       #212121      →      #E0E0E0 (不要纯白)
次要色:     #757575      →      #9E9E9E

深色模式中的高度 (Elevation):
├── 高度越高 = 表面颜色越浅 (Overlay)
├── 0dp →  0% 叠加
├── 4dp →  9% 叠加
└── 通过亮度区分层级，而不是阴影
```

### 颜色反转规则

```
❌ 不要 仅仅反转颜色:
├── 饱和色会变得刺眼
├── 语义色也会失去意义
├── 品牌色可能崩坏
└── 对比度变化不可预测

✅ 要 创建有意的深色色盘:
├── 降低主色饱和度
├── 使用更浅的色调 (Tints) 来强调
├── 保持语义色的含义
├── 独立检查对比度
```

## 4. 户外可见性

### 阳光问题

```
户外屏幕可见性:
├── 强光会冲淡低对比度内容
├── 眩光降低可读性
├── 偏光墨镜会有影响
└── 用户可能会用手遮挡屏幕
```

### 高对比度策略

```
为了户外可见性:

最小对比度比率:
├── 普通文字: 4.5:1 (WCAG AA)
├── 大号文字: 3:1 (WCAG AA)
├── 推荐: 7:1+ (AAA)

避免 (AVOID):
├── #999 在 #FFF 上 (未达标 AA)
├── 浅色在浅色背景上
└── 关键信息的微妙渐变

建议 (DO):
├── 使用系统语义色
├── 在强光环境下测试
├── 为关键 UI 使用实色
```

## 5. 语义色 (Semantic Colors)

### 一致的含义

| 语义               | 含义         | iOS 默认 | Android 默认 |
| :----------------- | :----------- | :------- | :----------- |
| **错误 (Error)**   | 问题, 破坏性 | #FF3B30  | #B3261E      |
| **成功 (Success)** | 完成, 积极   | #34C759  | #4CAF50      |
| **警告 (Warning)** | 注意, 谨慎   | #FF9500  | #FFC107      |
| **信息 (Info)**    | 信息         | #007AFF  | #2196F3      |

**规则:**

- **永远不要** 将语义色用于装饰或品牌 (会混淆含义)。
- **必须** 搭配图标使用 (照顾色盲用户)。

## 6. 动态取色 (Dynamic Color - Android)

### Material You

Android 12+ 的动态取色：
用户壁纸 → 提取颜色 → 生成 App 主题

- **支持动态取色**: 使用 Jetpack Compose `dynamicColorScheme()` 或 React Native 相关库。
- **降级 (Fallback)**: 必须提供一套静态的品牌色主题，以备不支持动态取色的设备或用户关闭该功能。

## 7. 色彩反模式

### ❌ 常见错误

| 错误                   | 问题            | 修复                 |
| :--------------------- | :-------------- | :------------------- |
| **浅灰在白色上**       | 户外不可见      | 最小 4.5:1 对比度    |
| **深色模式用纯白字**   | 眼睛疲劳 (光晕) | 使用 #E0E0E0-#F0F0F0 |
| **深色模式饱和度未减** | 刺眼, 荧光感    | 降低颜色饱和度       |
| **红/绿单一指示**      | 色盲用户不可用  | 增加图标辅助         |
| **忽略系统深色模式**   | 体验割裂        | 必须支持双模式       |

- [ ] **语义色符合惯例？**
- [ ] **色盲安全 (不依赖纯颜色)？**
- [ ] **在强阳光下测试过吗？**
- [ ] **在真实 OLED 设备上测试过深色模式吗？**

---

> **记住:** 移动端的色彩必须在最恶劣的条件下也能工作——强光、眼睛疲劳、色盲、低电量。如果在这些测试中失败，再漂亮的颜色也是无用的。
