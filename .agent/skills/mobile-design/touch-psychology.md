## 1. 触摸的菲茨定律 (Fitts' Law for Touch)

### 根本区别

```
桌面端 (鼠标/触控板):
├── 光标大小: 1 像素 (精准)
├── 视觉反馈: 悬停 (Hover) 状态
├── 错误成本: 低 (容易重试)
└── 目标获取: 快，精准

移动端 (手指):
├── 接触面积: ~7mm 直径 (不精准)
├── 视觉反馈: 无悬停，只有点击
├── 错误成本: 高 (令人沮丧的重试)
├── 遮挡: 手指会遮住目标
└── 目标获取: 较慢，需要更大的目标
```

### 菲茨定律适配公式

```
触摸获取时间 = a + b × log₂(1 + D/W)

其中:
├── D = 到目标的距离
├── W = 目标的宽度
└── 对于触摸: W 必须比桌面端大得多
```

### 最小触摸目标尺寸

| 平台                   | 最小        | 推荐    | 适用                 |
| :--------------------- | :---------- | :------ | :------------------- |
| **iOS (HIG)**          | 44pt × 44pt | 48pt+   | 所有可点击元素       |
| **Android (Material)** | 48dp × 48dp | 56dp+   | 所有可点击元素       |
| **WCAG 2.2**           | 44px × 44px | -       | 可访问性合规         |
| **关键操作**           | -           | 56-64px | 主要 CTA，破坏性操作 |

### 视觉尺寸 vs 热区 (Hit Area)

```
┌─────────────────────────────────────┐
│                                     │
│    ┌─────────────────────────┐      │
│    │                         │      │
│    │    [  BUTTON  ]         │ ← 视觉: 36px
│    │                         │      │
│    └─────────────────────────┘      │
│                                     │ ← 热区: 48px (padding 延伸)
└─────────────────────────────────────┘

✅ 正确: 如果热区最小为 44-48px，视觉元素可以较小
❌ 错误: 让热区等于较小的视觉元素
```

## 2. 拇指热区解剖 (Thumb Zone Anatomy)

### 单手手机使用

研究表明：**49%** 的用户单手握持手机。

### 拇指弧度 (右手用户)

```
右手握持手机:

┌───────────────────────────────┐
│  伸展 (STRETCH)    尚可       │
│                               │
│  伸展              尚可       │
│                               │
│     尚可          舒适 (EASY) │
│                               │
│    舒适           舒适        │
│                               │
│  舒适 (EASY)      舒适        │
│                               │
└───────────────────────────────┘
```

_左手镜像对应。_
**设计原则:** 为**双手**设计，或者假设右主导但兼顾左手。

### 放置指南

| 元素类型       | 理想位置       | 原因            |
| :------------- | :------------- | :-------------- |
| **主要 CTA**   | 底部中间/右侧  | 拇指容易够到    |
| **Tab bar**    | 底部           | 自然的拇指位置  |
| **FAB**        | 右下角         | 适合右手操作    |
| **导航**       | 顶部 (伸展区)  | 低频使用        |
| **破坏性操作** | 左上角         | 难够到 = 难误触 |
| **关闭/取消**  | 左上角         | 惯例 + 安全     |
| **确认/完成**  | 右上角 或 底部 | 惯例            |

### 大屏手机考量 (>6")

在超大手机上，顶部 40% 成为单手使用的"死区"。
**解决方案:**

- 下拉悬停 (Reachability, iOS)
- 下拉式界面 (抽屉把内容拉下来)
- 底部 Sheet 导航
- 使用手势替代顶部操作

## 3. 触摸 vs 点击心理学

### 期望差异

| 方面         | 点击 (Desktop) | 触摸 (Mobile)        |
| :----------- | :------------- | :------------------- |
| **反馈时机** | 可等待 100ms   | 期望即时 (<50ms)     |
| **视觉反馈** | 悬停 → 点击    | 即时点击响应         |
| **错误容忍** | 易重试         | 挫败感强，感觉像坏了 |
| **精确度**   | 高             | 低                   |
| **取消操作** | ESC 键         | 滑动取消，点击外部   |

### 触摸反馈需求

1.  **点击 → 即时视觉变化 (< 50ms)**
    - 高亮状态 (背景色变深/变浅)
    - 轻微缩小 (Scale down 0.95-0.98)
    - 涟漪效果 (Android Material)
    - 触觉反馈 (Haptics) 确认
    - **绝对不要** 毫无反应！

2.  **加载 → 100ms 内显示**
    - 如果操作超过 100ms
    - 显示加载器/进度
    - 禁用按钮 (防止双击)
    - 尽可能使用乐观 UI (Optimistic UI)

### "胖手指"问题 (Fat Finger)

**问题:** 手指在点击时遮挡了目标。

- 用户看不清点到了哪里。
- 视觉反馈出现在手指**下方**。

**解决方案:**

- **上方反馈**: 在触摸点上方显示反馈 (Tooltips)。
- **偏移**: 对于精准任务使用光标偏移。
- **放大镜**: 文本选择时的放大镜。
- **大目标**: 目标足够大，不需要精准点击。

## 4. 手势心理学

### 手势可发现性问题

**问题:** 手势是**不可见**的。

- 用户必须发现/记住它们。
- 没有悬停提示。
- 许多用户永远发现不了手势。

**解决方案:** 始终提供可见的替代方案。

- 左滑删除 → 也要显示删除按钮或菜单。
- 下拉刷新 → 也要显示刷新按钮。
- 手势只作为**快捷方式**，而不是唯一途径。

### 常见手势惯例

| 手势                 | 通用含义   | 用法       |
| :------------------- | :--------- | :--------- |
| **Tap**              | 选择, 激活 | 主要操作   |
| **Double tap**       | 放大, 点赞 | 快速操作   |
| **Long press**       | 上下文菜单 | 次要选项   |
| **Swipe horizontal** | 导航, 删除 | 列表操作   |
| **Swipe down**       | 刷新, 关闭 | 下拉刷新   |
| **Pinch**            | 缩放       | 地图, 图片 |

### 手势示能 (Affordance) 设计

滑动操作需要视觉提示：

- 边缘提示 (部分颜色露出)
- 拖拽手柄图标 ( ≡ ) 提示可排序
- 引导页 Tooltip 解释手势
- ❌ **坏:** 没有任何视觉示能的隐藏手势

## 5. 触觉反馈模式 (Haptic Feedback)

### 为什么触觉很重要

- 提供**确认感**，无需用眼看。
- 提供更**高级/Premium** 的质感。
- 辅助功能 (盲人用户)。
- 减少错误率。
- **没有触觉:** 感觉廉价、像网页、不确定是否点击成功。

### 触觉类型 (iOS/Android)

| 类型                   | 用途                              |
| :--------------------- | :-------------------------------- |
| **Light / Selection**  | 滚动选择器, Toggle 开关, 轻微交互 |
| **Medium / Click**     | 标准点击确认                      |
| **Heavy / Long Press** | 重要操作完成, 掉落, 长按          |
| **Success**            | 任务成功完成                      |
| **Warning**            | 警告, 需要注意                    |
| **Error**              | 错误发生                          |

### 触觉使用指南

✅ **使用触觉:**

- 按钮点击
- 开关切换
- 选择器滚动
- 下拉刷新触发
- 成功/错误/警告状态
- 滑动操作阈值

❌ **不要使用触觉:**

- 每个滚动位置 (太频繁)
- 每个列表项
- 被动显示
- 太频繁 (导致触觉疲劳)

## 6. 移动端认知负荷

### 减少移动端认知负荷

1.  **单一主要操作**: 每屏一个焦点。
2.  **渐进式披露**: 只显示当前需要的。
3.  **智能默认值**: 预填能预填的。
4.  **分块 (Chunking)**: 长表单分步。
5.  **识别重于回忆**: 给出选项，别让用户背诵。
6.  **上下文持久化**: 中断后保存状态。

### 米勒定律移动版 (Miller's Law)

- 桌面: 7±2 个项目
- **移动端: 5±1 (更多干扰)**
    - 导航: 最多 5 个 Tab
    - 选项: 每级菜单最多 5 个
    - 步骤: 进度条最多 5 步

## 7. 触摸可访问性

### 运动障碍考量

- 可能有震颤 (需要大目标)。
- 使用辅助设备。
- 射程有限 (单手必要性)。
- 需要更多时间。

**设计响应:**

- 慷慨的触摸目标 (48dp+)。
- 可调节的手势时间。
- 破坏性操作提供撤销。

### WCAG 2.2 触摸目标间距

- 宽度/高度 ≥ 44px
- 或者 间距 ≥ 8px
- 除非是文本内联链接或用户可控

## 8. 触摸中的情感

### 高级感 (Premium Feel)

什么让触摸感觉"高级"：

- 即时响应 (< 50ms)
- 恰当的触觉反馈
- 流畅的 60fps/120fps 动画
- 正确的阻尼/物理效果
- 声音反馈 (适当时)

### 信任构建

触摸交互中的信任信号：

- **一致性**: 相同动作 = 相同反应。
- **可靠性**: 从不静默失败。
- **安全感**: 敏感操作有阻力/确认。
