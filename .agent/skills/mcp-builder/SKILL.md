# MCP 构建器

> 构建 MCP 服务器的原则。

## 1. MCP 概述

### 什么是 MCP？(What is MCP?)

模型上下文协议 (Model Context Protocol) - 连接 AI 系统与外部工具和数据源的标准。

### 核心概念

| 概念                 | 用途               |
| -------------------- | ------------------ |
| **Tools (工具)**     | AI 可以调用的函数  |
| **Resources (资源)** | AI 可以读取的数据  |
| **Prompts (提示词)** | 预定义的提示词模板 |

---

## 3. 工具设计原则

### 良好的工具设计

| 原则           | 描述                                |
| -------------- | ----------------------------------- |
| 清晰的名称     | 面向动作 (get_weather, create_user) |
| 单一目的       | 做好一件事                          |
| 经过验证的输入 | 带有类型和描述的 Schema (模式)      |
| 结构化输出     | 可预测的响应格式                    |

### 输入 Schema 设计

| 字段               | 必填？       |
| ------------------ | ------------ |
| Type (类型)        | 是 - object  |
| Properties (属性)  | 定义每个参数 |
| Required (必填)    | 列出强制参数 |
| Description (描述) | 人类可读     |

---

## 4. 资源模式

### 资源类型

| 类型            | 用途                  |
| --------------- | --------------------- |
| Static (静态)   | 固定数据 (配置, 文档) |
| Dynamic (动态)  | 请求时生成            |
| Template (模板) | 带参数的 URI          |

### URI 模式

| 模式                   | 示例                |
| ---------------------- | ------------------- |
| Fixed (固定)           | `docs://readme`     |
| Parameterized (参数化) | `users://{userId}`  |
| Collection (集合)      | `files://project/*` |

---

## 5. 错误处理

### 错误类型

| 情况       | 响应                        |
| ---------- | --------------------------- |
| 无效参数   | 验证错误消息                |
| 未找到     | 清晰的 "not found (未找到)" |
| 服务器错误 | 通用错误，记录详细信息      |

### 最佳实践

- 返回结构化错误
- 不要暴露内部细节
- 记录日志以便调试
- 提供可操作的消息

---

## 6. 多模态处理

### 支持的类型

| 类型          | 编码               |
| ------------- | ------------------ |
| Text (文本)   | 纯文本             |
| Images (图像) | Base64 + MIME type |
| Files (文件)  | Base64 + MIME type |

---

## 7. 安全原则

### 输入验证

- 验证所有工具输入
- 对用户提供的数据进行脱敏
- 限制资源访问

### API 密钥

- 使用环境变量
- 不要记录密钥
- 验证权限
