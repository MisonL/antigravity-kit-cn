# 头脑风暴与需求澄清 (Brainstorming)

此技能用于 `/brainstorm` 工作流和 **Socratic Gate (苏格拉底之门)**。
在 `/debug`、`/test` 等高风险流程前，也可用于补齐上下文与边界条件。

## 苏格拉底之门 (Socratic Gate)

**规则**: 在写任何代码之前，如果需求不明确，必须先提问。

### 用法

当用户提出："帮我做一个网站" (太模糊)
AI 回应：

1. "这个网站的目标受众是谁？"
2. "你需要展示哪些核心数据？"
3. "它是静态展示还是有复杂交互？"

## 引导技巧

1.  **探寻动机 (Why)**: "为什么需要这个功能？是为了解决什么问题？"
2.  **挑战假设 (Challenge)**: "我们真的需要引入这个库吗？原生实现是否足够？"
3.  **探索边界 (Edge Cases)**: "如果数据量达到 100万 会怎样？"

## 输出格式

头脑风暴结束后，应整理为结构化的文档：

- **Goal (目标)**
- **Scope (范围)**
- **Key Features (核心功能)**
- **Tech Stack (技术栈)**
