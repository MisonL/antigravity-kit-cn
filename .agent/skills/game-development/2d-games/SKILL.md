# 2D 游戏开发

> 2D 游戏系统原则。

---

## 1. 精灵系统

### 精灵组织

| 组件 | 用途 |
| --- | --- |
| **Atlas（图集）** | 合并纹理，减少 draw calls（绘制调用） |
| **Animation（动画）** | 帧序列 |
| **Pivot（轴心）** | 旋转/缩放原点 |
| **Layering（分层）** | Z-order 控制 |

### 动画原则

- 帧率：典型 8-24 FPS
- Squash and stretch（挤压与拉伸）增强冲击力
- Anticipation（预备动作）
- Follow-through（动作跟随）

---

## 2. 瓦片地图设计

### 瓦片考量

| 因素 | 建议 |
| --- | --- |
| **尺寸** | 16x16, 32x32, 64x64 |
| **Auto-tiling（自动瓦片）** | 用于地形 |
| **碰撞** | 简化形状 |

### 图层

| 图层 | 内容 |
| --- | --- |
| Background（背景） | 非交互式风景 |
| Terrain（地形） | 可行走的地面 |
| Props（道具） | 交互式对象 |
| Foreground（前景） | 视差覆盖层 |

---

## 3. 2D 物理

### 碰撞形状

| 形状 | 适用场景 |
| --- | --- |
| Box（盒） | 矩形物体 |
| Circle（圆） | 球体、圆形物体 |
| Capsule（胶囊） | 角色 |
| Polygon（多边形） | 复杂形状 |

### 物理考量

- 像素完美 vs 基于物理
- 固定时间步长以保持一致性
- 使用分层进行过滤

---

## 4. 相机系统

### 相机类型

| 类型 | 用途 |
| --- | --- |
| **Follow（跟随）** | 追踪玩家 |
| **Look-ahead（前瞻）** | 预测移动 |
| **Multi-target（多目标）** | 双人游戏 |
| **Room-based（基于房间）** | Metroidvania |

### 屏幕震动

- 短持续时间（50-200ms）
- 强度递减
- 适度使用

---

## 5. 类型模式

### 平台游戏

- Coyote time（土狼时间，边缘后的宽容度）
- Jump buffering（跳跃缓冲）
- 可变跳跃高度

### 俯视视角

- 8 方向或自由移动
- 基于瞄准或自动瞄准
- 考虑是否旋转

---

## 6. 反模式

| ❌ 禁止 | ✅ 推荐 |
| --- | --- |
| 分离纹理 | 使用图集 |
| 复杂碰撞形状 | 简化碰撞 |
| 抖动相机 | 平滑跟随 |
| 物理上追求像素完美 | 选择一种方法 |

---

> **记住：** 2D 的核心是清晰度。每个像素都应传达信息。
