# 文档模板 (Documentation Templates)

> 常用文档类型的模板与结构指南。

---

## 1. README 结构

### 核心章节 (按优先级排序)

| 章节                        | 目的                 |
| --------------------------- | -------------------- |
| **标题 + 一句话简介**       | 这是什么？           |
| **快速上手 (Quick Start)**  | 在 <5 分钟内运行起来 |
| **功能特性 (Features)**     | 能做什么？           |
| **配置 (Configuration)**    | 如何自定义           |
| **API 参考**                | 链接到详细文档       |
| **贡献指南 (Contributing)** | 如何参与开发         |
| **许可证 (License)**        | 法律条款             |

### README 模板

```markdown
# 项目名称

简明扼要的一句话描述。

## 快速上手

[运行该项目的最简步骤]

## 功能特性

- 功能 1
- 功能 2

## 配置

| 变量名 | 描述       | 默认值 |
| ------ | ---------- | ------ |
| PORT   | 服务器端口 | 3000   |

## 相关文档

- [API 参考](./docs/api.md)
- [架构设计](./docs/architecture.md)

## 许可证

MIT
```

## 3. 代码注释指南

### JSDoc/TSDoc 模板

```typescript
/**
 * 简要描述函数的功能。
 *
 * @param paramName - 参数描述
 * @returns 返回值描述
 * @throws ErrorType - 何时会抛出此错误
 *
 * @example
 * const result = functionName(input);
 */
```

### 何时应该写注释

| ✅ 应该写注释           | ❌ 不建议写注释               |
| ----------------------- | ----------------------------- |
| 解释“为什么” (业务逻辑) | 解释“是什么” (显而易见的代码) |
| 复杂的算法逻辑          | 每一行都写注释                |
| 非直观的行为            | 自解释的代码                  |
| API 契约/约定           | 内部实现细节                  |

## 5. 架构决策记录 (ADR)

```markdown
# ADR-001: [决策标题]

## 状态

已采纳 (Accepted) / 已废弃 (Deprecated) / 已被替代 (Superseded)

## 背景

我们为什么要做出这个决定？当时的上下文是什么？

## 决策

我们最终决定了什么？

## 后果

这一决策带来了哪些权衡 (Trade-offs)？
```

## 6. AI 友好文档 (2025)

### llms.txt 模板

针对 AI 爬虫和 Agent 优化的说明文件：

```markdown
# 项目名称

> 项目目标的一句话概述。

## 核心文件

- [src/index.ts]: 主入口
- [src/api/]: API 路由
- [docs/]: 文档目录

## 关键概念

- 概念 1: 简要解释
- 概念 2: 简要解释
```

### 适配 MCP 的文档规范

为了便于 RAG (检索增强生成) 索引：

- 清晰的 H1-H3 层级
- 针对数据结构的 JSON/YAML 示例
- 针对流程的 Mermaid 图表
- 自包含的章节内容

## 7. 结构原则 (Structure Principles)

| 原则                          | 原因                           |
| ----------------------------- | ------------------------------ |
| **可扫描性 (Scannable)**      | 使用标题、列表、表格方便查阅   |
| **示例优先 (Examples first)** | 展示而非仅仅陈述               |
| **渐进式描述**                | 从简单到复杂                   |
| **保持更新**                  | 过时的文档比没有文档更具误导性 |

> **记住：** 模板只是起点。请根据您项目的实际需求进行灵活调整。
