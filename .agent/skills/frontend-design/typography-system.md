# 排版系统参考 (Typography System Reference)

> 排版原则与决策制定 - 学习思考，而非死记。
> **没有固定的字体名称或尺寸 - 理解系统。**

---

## 1. 模块化尺度原则 (Modular Scale Principles)

### 什么是模块化尺度？

```
字体尺寸之间的数学关系:
├── 选择一个 基础 (BASE) 尺寸 (通常是正文)
├── 选择一个 比率 (RATIO) (倍数)
└── 使用公式生成所有尺寸: base × ratio^n
```

### 常用比率与适用场景

| 比率                    | 值    | 感觉       | 适用              |
| :---------------------- | :---- | :--------- | :---------------- |
| 小二度 (Minor Second)   | 1.067 | 极其微妙   | 密集 UI, 小屏幕   |
| 大二度 (Major Second)   | 1.125 | 微妙       | 紧凑界面          |
| 小三度 (Minor Third)    | 1.2   | 舒适       | 移动应用, 卡片    |
| 大三度 (Major Third)    | 1.25  | 平衡       | 通用网页 (最常用) |
| 纯四度 (Perfect Fourth) | 1.333 | 明显       | 编辑, 博客        |
| 纯五度 (Perfect Fifth)  | 1.5   | 戏剧性     | 标题, 营销        |
| 黄金比 (Golden Ratio)   | 1.618 | 最大冲击力 | Hero 区块, 展示   |

### 生成你的尺度

```
给定: base = 你的基础尺寸, ratio = 你的比率

Scale:
├── xs:  base ÷ ratio²
├── sm:  base ÷ ratio
├── base: 你的基础尺寸
├── lg:  base × ratio
├── xl:  base × ratio²
├── 2xl: base × ratio³
├── 3xl: base × ratio⁴
└── ... 按需继续
```

## 2. 字体搭配原则

### 为什么字体能搭配在一起

```
对比 + 和谐:
├── 足够不同 以创造层级
├── 足够相似 以感觉协调
└── 通常: 衬线 + 无衬线, 或 展示 + 中性
```

### 搭配策略

| 策略         | 如何做                  | 结果         |
| :----------- | :---------------------- | :----------- |
| **对比**     | 衬线标题 + 无衬线正文   | 经典, 编辑感 |
| **同家族**   | 一个可变字体, 不同字重  | 协调, 现代   |
| **同设计师** | 同一 Foundry 出品的字体 | 通常比例和谐 |
| **年代匹配** | 同一历史时期的字体      | 历史一致性   |

### 寻找什么

```
搭配时，比较:
├── x-height (小写字母高度)
├── 字母宽度 (窄 vs 宽)
├── 笔画对比 (粗/细变化)
└── 整体情绪 (正式 vs 休闲)
```

## 3. 行高原则 (Line Height)

### 关系

```
行高取决于:
├── 字号 (字号越大 = 需要的行高比例越小)
├── 行长 (行越长 = 需要越多行高)
├── 字体设计 (某些字体需要更多空间)
└── 内容类型 (标题 vs 正文)
```

### 上下文指南

| 内容类型             | 行高范围  | 为什么         |
| :------------------- | :-------- | :------------- |
| **标题 (Headings)**  | 1.1 - 1.3 | 行短，需要紧凑 |
| **正文 (Body)**      | 1.4 - 1.6 | 舒适阅读       |
| **长文 (Long-form)** | 1.6 - 1.8 | 最大易读性     |
| **UI 元素**          | 1.2 - 1.4 | 空间效率       |

---

## 4. 行长原则 (Line Length)

### 最佳阅读宽度

```
甜蜜点: 每行 45-75 个字符
├── < 45: 太碎，打断心流
├── 45-75: 舒适阅读
├── > 75: 眼球追踪疲劳
```

### 如何测量

```css
/* 基于字符 (推荐) */
max-width: 65ch; /* ch = "0" 字符的宽度 */

/* 这会自动适应字号 */
```

## 5. 响应式排版原则

### 问题

```
固定尺寸无法很好地缩放:
├── 桌面尺寸在手机上太大
├── 手机尺寸在桌面上太小
└── 断点跳跃感觉突兀
```

### 流体排版 (clamp)

```css
/* 语法: clamp(最小, 首选, 最大) */
font-size: clamp(MINIMUM_SIZE, FLUID_CALCULATION, MAXIMUM_SIZE);
```

### 缩放策略

| 元素     | 缩放行为                   |
| :------- | :------------------------- |
| 正文     | 轻微缩放 (1rem → 1.125rem) |
| 副标题   | 中度缩放                   |
| 标题     | 更具戏剧性的缩放           |
| 展示大字 | 最剧烈的缩放               |

## 6. 字重与强调原则

### 语义化字重

| 字重范围 | 名称         | 用途            |
| :------- | :----------- | :-------------- |
| 300-400  | Light/Normal | 正文, 段落      |
| 500      | Medium       | 微妙强调        |
| 600      | Semibold     | 副标题, 标签    |
| 700      | Bold         | 标题, 强强调    |
| 800-900  | Heavy/Black  | 展示, Hero 文字 |

### 创造对比

```
好对比 = 跳过至少 2 个字重级别
├── 400 正文 + 700 标题 = 好
├── 400 正文 + 500 强调 = 微妙
├── 600 标题 + 700 副标题 = 太相似
```

## 7. 字间距 (Tracking)

### 原则

```
大号字 (标题): 紧缩字间距
├── 字母大，间隙感觉更大
└── 轻微负值看起来更好

小号字 (正文): 正常或稍宽
├── 在小尺寸下提高易读性
└── 正文绝不要用负值

全大写 (ALL CAPS): 总是加宽
├── 大写字母缺乏升部/降部
└── 需要更多空间才感觉对
```

## 8. 层级原则

### 通过排版建立视觉层级

```
建立层级的方法:
├── 尺寸 (SIZE) (最明显)
├── 字重 (WEIGHT) (粗体突出)
├── 颜色 (COLOR) (对比度层级)
├── 间距 (SPACING) (边距分隔区域)
└── 位置 (POSITION) (顶部 = 重要)
```

### 典型层级

| 级别          | 特征                   |
| :------------ | :--------------------- |
| 主标题 (H1)   | 最大, 最粗, 最独特     |
| 二级标题 (H2) | 明显较小但仍粗         |
| 三级标题 (H3) | 中等大小, 可能仅用字重 |
| 正文 (Body)   | 标准尺寸和字重         |
| 说明/元数据   | 更小, 通常颜色更浅     |

## 9. 易读性心理学

### F型阅读模式

```
用户以 F 型扫描:
├── 扫过顶部 (第一行)
├── 沿左侧向下
├── 再次横扫 (副标题)
└── 继续沿左侧向下
```

**启示**: 关键信息放在左侧和标题中。

### 信息分块

- 短段落 (最多 3-4 行)
- 清晰的副标题
- 列表项使用项目符号
- 板块之间留白

## 10. 排版选择检查清单

在最终确定排版之前：

- [ ] **询问了用户字体偏好？**
- [ ] **考虑了品牌/上下文？**
- [ ] **选择了合适的比例？**
- [ ] **限制在 2-3 个字体家族？**
- [ ] **测试了所有尺寸的易读性？**
- [ ] **检查了行长 (45-75ch)？**
- [ ] **验证了对比度 (无障碍)？**
