# 排版系统参考 (Typography System Reference)

> 排版原则与决策 —— 学会思考，而不是死记硬背。
> **没有必须死记硬背的固定字体名或字号 —— 请理解系统背后的逻辑。**

---

## 1. 模块化比例原则 (Modular Scale)

### 什么是模块化比例？

```
字体大小之间的数学关系：
├── 确定一个 基准 (BASE) 大小 (通常是正文字号)
├── 确定一个 比例 (RATIO) (乘数)
└── 使用公式生成所有字号: base × ratio^n
```

### 常见比例及适用场景

| 比例名称                  | 数值  | 感受       | 最佳适用              |
| ------------------------- | ----- | ---------- | --------------------- |
| 小二度 (Minor Second)     | 1.067 | 极细微     | 密集型 UI, 小屏幕     |
| 大二度 (Major Second)     | 1.125 | 微妙       | 紧凑型界面            |
| 小三度 (Minor Third)      | 1.2   | 舒适       | 移动端 App, 卡片布局  |
| 大三度 (Major Third)      | 1.25  | 均衡       | 通用网页 (最常用)     |
| 完全四度 (Perfect Fourth) | 1.333 | 显著       | 社论、博客            |
| 完全五度 (Perfect Fifth)  | 1.5   | 戏剧化     | 大标题、营销页面      |
| 黄金比例 (Golden Ratio)   | 1.618 | 冲击力极强 | Hero 区域、展示型页面 |

### 如何生成比例尺

```
已知: 基准 = BASE_SIZE, 比例 = RATIO

字号序列:
├── xs:  base ÷ ratio²
├── sm:  base ÷ ratio
├── base: 基准字号 (通常 16px)
├── lg:  base × ratio
├── xl:  base × ratio²
├── 2xl: base × ratio³
├── 3xl: base × ratio⁴
└── ... 根据需要继续
```

### 确定基准字号 (Base Size)

| 背景       | 基准字号范围 | 原因                 |
| ---------- | ------------ | -------------------- |
| 移动端优先 | 16-18px      | 确保小屏幕上的易读性 |
| 桌面端应用 | 14-16px      | 提高信息密度         |
| 社论/博客  | 18-21px      | 提升长文阅读的舒适度 |
| 侧重无障碍 | 18px+        | 更容易阅读           |

## 2. 字体搭配原则 (Font Pairing)

### 什么样的字体能搭在一起？

```
对比 + 和谐：
├── 差异足够大，以建立清晰的层级 (Hierarchy)
├── 相似点足够多，以保持视觉凝聚力 (Cohesion)
└── 常见模式: 衬线标题 + 无衬线正文，或 展示字体 + 中性字体
```

### 搭配策略

| 策略           | 方式                             | 效果             |
| -------------- | -------------------------------- | ---------------- |
| **对比**       | 衬线标题 + 无衬线正文            | 经典、社论感     |
| **同族**       | 选用同一种可变字体，使用不同字重 | 极度统一、现代感 |
| **同名设计师** | 选用出自同一设计工作室的字体     | 比例通常非常和谐 |
| **时代契合**   | 选用同一历史时期的字体           | 历史风格的一致性 |

### 避坑指南

- ❌ 两个装饰性极强的字体搭在一起。
- ❌ 两个非常相似但细节有冲突的字体。
- ❌ 在同一个页面使用超过 2-3 个字体家族。
- ❌ 两个 x-height (小写字母高度) 差异极大的字体。

## 3. 行高原则 (Line Height)

### 关联关系

```
行高取决于：
├── 字号 (字号越大，所需的行高相对比例越小)
├── 行长 (行越长，需要的行高越大，方便视线换行)
├── 字体设计 (有些字体本身需要更多的呼吸空间)
└── 内容类型 (标题 vs 正文)
```

### 不同背景的指南

| 内容类型     | 行高范围  | 原因             |
| ------------ | --------- | ---------------- |
| **标题**     | 1.1 - 1.3 | 行短，追求紧凑感 |
| **正文**     | 1.4 - 1.6 | 舒适的日常阅读   |
| **长文内容** | 1.6 - 1.8 | 追求最高易读性   |
| **UI 元素**  | 1.2 - 1.4 | 提高空间利用率   |

---

## 4. 行长原则 (Line Length)

### 最佳阅读宽度

```
最佳区间：每行 45-75 个字符
├── < 45: 过于破碎，打断阅读流
├── 45-75: 舒适的阅读节奏
├── > 75: 视线换行时易产生追踪疲劳
```

## 5. 响应式排版原则

### 解决方案：流体排版 (clamp)

```css
/* 语法: clamp(最小值, 理想值, 最大值) */
font-size: clamp(1rem, 2vw + 1rem, 3rem);
```

## 6. 字重与强调原则 (Weight)

| 字重范围 | 名称         | 适用场景       |
| -------- | ------------ | -------------- |
| 300-400  | Light/Normal | 正文, 段落     |
| 500      | Medium       | 微弱的强调     |
| 600      | Semibold     | 小标题, 标签   |
| 700      | Bold         | 标题, 强力强调 |
| 800-900  | Heavy/Black  | Hero 展示文字  |

---

## 7. 字间距 (Tracking)

### 原则

```
大文字 (标题): 调紧间距
├── 文字越大，字母间的裂缝感越明显
├── 使用轻微的负间距 (Negative tracking) 效果更好

小文字 (正文): 正常或轻微调宽
├── 提高小字号下的辨识度

全大写 (ALL CAPS): 必须调宽间距
├── 全大写字母缺乏上下边缘的变化，需要更多空间来平衡
```

---

## 8. 层级原则 (Hierarchy)

### 建立层级的方式：

- **字号** (最直观)。
- **字重** (粗体更引人注意)。
- **颜色** (使用不同的对比度)。
- **间距** (通过边距分隔不同章节)。

---

## 9. 阅读心理学

### F-Pattern 阅读模式

用户通常呈 F 型扫描：先看顶部第一行，再沿左侧向下，然后看子标题，最后继续沿左侧向下。
**启示**: 关键信息应放在左侧和标题中。

## 10. 排版选择检查清单

在最终确定排版前：

- [ ] **询问了用户字体的偏好？**
- [ ] **考虑了品牌调性与项目背景？**
- [ ] **选择了合适的比例尺系数？**
- [ ] **字体家族是否限制在 2-3 个以内？**
- [ ] **测试了所有关键尺寸下的易读性？**
- [ ] **检查了行长 (控制在 45-75ch 之间)？**
- [ ] **验证了无障碍对比度是否达标？**
- [ ] **是否不同于你上一个项目使用的字体？**

### 反模式 (应避免的做法)

- ❌ 每个项目都用同样的字体。
- ❌ 使用过多的字体家族。
- ❌ 为了所谓的“设计感”而牺牲易读性。
- ❌ 使用固定尺寸而不考虑响应式缩放。
- ❌ 在正文中使用装饰性过强的字体。

---

> **记住**: 排版是为了沟通的清晰度。根据内容需求和受众来选择，而不是你的个人喜好。
