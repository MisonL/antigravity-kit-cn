# 决策树与背景模板 (Decision Trees & Context Templates)

> 基于背景的设计，而不是固定死板的方案。
> **这些是决策指南，而非可以直接复制使用的模板。**
> **关于 UX 心理学原则 (席克定律、菲茨定律等) 请参阅:** [ux-psychology.md](ux-psychology.md)

---

## ⚠️ 如何使用此文件

此文件是为了辅助你进行 **决策**。

- 决策树 → 引导你思考不同的选项及其背后的逻辑。
- 模板 → 展示设计 **结构** 和 **原则**，而非具体的数值。
- **在实施前，始终先询问用户的偏好。**
- **根据具体背景生成新鲜的色盘**，不要直接复制 Hex 代码。
- **应用 UX 定律** (来自 ux-psychology.md) 来验证你的决策是否合理。

## 2. 受众决策树 (Audience Decision Tree)

### 谁是你的目标用户？

```
目标受众 (Target Audience)
      │
      ├── Z 世代 (18-25)
      │   ├── 颜色: 大胆、鲜艳、意想不到的配色组合
      │   ├── 字体: 巨型排版、表现力强、可变字体
      │   ├── 布局: 移动端优先、纵向流、碎片化内容
      │   ├── 动效: 强动感、游戏化、深度交互
      │   └── 策略: 追求真实感、快节奏、拒绝排版死板的企业感
      │
      ├── 千禧一代 (26-41)
      │   ├── 颜色: 柔和、大地色系、具备高级感
      │   ├── 字体: 整洁、易读、强调功能性
      │   ├── 布局: 响应式、卡片化、组织井然有序
      │   ├── 动效: 微妙且有明确目的的反馈
      │   └── 策略: 价值驱动、透明化、可持续感
      │
      ├── X 世代 (42-57)
      │   ├── 颜色: 专业、值得信赖、保守色系
      │   ├── 字体: 熟悉感强的字体、清晰、务实
      │   ├── 布局: 传统层级结构、可预测的操作逻辑
      │   ├── 动效: 极简、仅限功能性反馈
      │   └── 策略: 直接、高效、强调可靠性
      │
      ├── 婴儿潮一代 (58+)
      │   ├── 颜色: 高对比度、简单明了
      │   ├── 字体: 字号大、极高易读性
      │   ├── 布局: 简单的线性布局、不杂乱
      │   ├── 动效: 无动画或极少动画
      │   └── 策略: 表达清晰、细节详实、值得信赖
      │
      └── B2B / 企业级
          ├── 颜色: 专业调色板、克制且沉稳
          ├── 字体: 整洁、对数据友好、易于扫描查阅
          ├── 布局: 基于网格、组织严密、操作高效
          ├── 动效: 专业感、微妙的过渡
          └── 策略: 专家导向、解决方案聚焦、ROI 驱动
```

## 3. 色彩选择决策树

### 不要锁定 Hex 代码，请遵循以下流程：

```
您想要传达什么情绪/引导什么行为？
            │
            ├── 信任与安全感 (Trust & Security)
            │   └── 考虑: 蓝色系, 专业的中性色
            │       → 询问用户是否有具体深浅偏好
            │
            ├── 成长与健康 (Growth & Health)
            │   └── 考虑: 绿色系, 大地色调
            │       → 询问用户是否侧重环保/自然/康养
            │
            ├── 紧迫感与行动 (Urgency & Action)
            │   └── 考虑: 暖色调 (橙/红) 作为强调色 (ACCENTS)
            │       → 谨慎使用, 询问强度是否合适
            │
            ├── 奢华与高端 (Luxury & Premium)
            │   └── 考虑: 极深色调, 金属质感, 克制的配色范围
            │       → 询问品牌定位与受众
            │
            ├── 创意与活泼 (Creative & Playful)
            │   └── 考虑: 多色彩, 意想不到的撞色方案
            │       → 询问品牌性格倾向
            │
            └── 冷静与极简 (Calm & Minimal)
                └── 考虑: 纯中性色配合单一转折色
                    → 询问哪个强调色契合品牌
```

### 执行流程：

1. 明确所需的情绪。
2. 缩小到特定的 **色系 (FAMILY)**。
3. **询问用户** 在该色系内的偏好。
4. 应用 HSL 原则生成新鲜的色盘。

## 4. 排版决策树 (Typography)

```
内容类型是什么？
          │
          ├── 数据密集型 (仪表盘, SaaS 指标)
          │   ├── 风格: 无衬线体 (Sans-serif), 清晰, 紧凑
          │   ├── 比例: 较窄的缩放比例 (1.125-1.2)
          │   └── 优先级: 易查阅性, 较高的信息密度
          │
          ├── 社论/内容类 (博客, 杂志)
          │   ├── 风格: 衬线体标题 + 无衬线正文 (经典搭配)
          │   ├── 比例: 较有张力的缩放比例 (1.333+)
          │   └── 优先级: 阅读舒适度, 视觉层级
          │
          ├── 现代科技 (初创公司, SaaS 营销页)
          │   ├── 风格: 几何型或人文主义无衬线体
          │   ├── 比例: 均衡比例 (1.25)
          │   └── 优先级: 现代感, 表达清晰
          │
          ├── 奢侈品 (时尚, 高端品牌)
          │   ├── 风格: 优雅的衬线体或极细的无衬线体
          │   ├── 比例: 极具戏剧化的比例 (1.5-1.618)
          │   └── 优先级: 这种地方的奢华体现为大量的留白
          │
          └── 趣味/休闲 (儿童, 游戏, 社交)
              ├── 风格: 圆润、友善的字体
              ├── 比例: 多样化、富有表现力
              └── 优先级: 有趣、亲近感、易于阅读
```

### 选择流程：

1. 确定内容类型。
2. 选择风格 **方向 (DIRECTION)**。
3. **询问用户** 是否有现成的品牌字体。
4. 选择符合方向的字体。

## 5. 电子商务指南 (E-commerce)

### 关键原则 (非硬性规定)

- **信任第一:** 如何展示安全性？
- **行动导向:** CTA 按钮放在哪里？
- **易于扫描:** 用户能快速对比产品吗？

### 颜色思考：

```
电商通常需要：
├── 信任色 (通常是蓝色系) → 询问偏好
├── 整洁的背景 (白色/中性色) → 取决于品牌调性
├── 行动强调色 (用于 CTA, 促销) → 取决于紧迫程度
├── 语义化的成功/错误提示 → 使用标准约定即可
└── 品牌集成 → 询问是否有现有色值
```

### 布局原则：

```
┌────────────────────────────────────────────────────┐
│  页眉 (HEADER): 品牌 + 搜索 + 购物车                   │
│  (保持核心操作始终可见)                                │
├────────────────────────────────────────────────────┤
│  信任区: 为什么要信任这个网站？                         │
│  (运费、退换、安全保障 - 如果适用)                      │
├────────────────────────────────────────────────────┤
│  主要展示区 (HERO): 核心信息或促销活动                  │
│  (清晰的单个 CTA)                                     │
├────────────────────────────────────────────────────┤
│  分类导航: 简便的导航路径                               │
│  (视觉化、可过滤、易扫描)                               │
├────────────────────────────────────────────────────┤
│  产品展示: 易于比较                                     │
│  (价格、评分、快捷操作可见)                             │
├────────────────────────────────────────────────────┤
│  社交证明: 别人为什么信任                               │
│  (评论、客户证言 - 如果有的话)                          │
├────────────────────────────────────────────────────┤
│  页脚 (FOOTER): 所有的细节说明                          │
│  (政策、联系方式、信任徽章)                             │
└────────────────────────────────────────────────────┘
```

## 6. SaaS 仪表盘指南 (SaaS Dashboard)

### 关键原则

- **功能第一:** 数据的清晰度重过装饰。
- **冷静的界面:** 降低认知负荷。
- **一致性:** 使用可预测的设计模式。

### 颜色思考：

```
仪表盘通常需要：
├── 背景: 浅色 或 深色 (询问用户偏好)
├── 表面 (Surface): 与背景有轻微对比
├── 主强调色: 用于关键操作
├── 数据颜色: 语义化的 成功/警告/危险 颜色
└── 暗淡色: 用于次要信息
```

### 布局原则：

```
考虑以下模式 (非强制):

模式 A: 侧边栏 + 内容
├── 固定侧边栏用于主导航
└── 中心区域用于展示内容

模式 B: 顶部导航 + 内容
├── 水平导航栏
└── 提供更宽的内容展示空间

模式 C: 折叠 + 可扩展
├── 仅图标的侧边栏可展开
└── 最大化内容展示区域

→ 询问用户对导航模式的偏好
```

## 7. 落地页指南 (Landing Page)

### 关键原则

- **以 Hero 为中心:** 第一印象最重要。
- **目标单一:** 只有一个主 CTA 引导。
- **情感连接:** 在尝试销售前先建立共鸣。

### 颜色思考：

```
落地页通常需要：
├── 品牌主色: 用于 Hero 背景或强调
├── 清爽的辅助色: 覆盖页面大部分区域
├── 突出的 CTA 颜色: 必须在全场脱颖而出
├── 支撑色: 用于不同章节或证言区域
└── 务必先询问品牌配色！
```

## 9. 设计前检查清单

### 在开始任何设计前

- [ ] **受众明确吗？** (具体是谁)
- [ ] **核心目标确定吗？** (引导什么行为)
- [ ] **已知约束条件吗？** (时间、品牌、技术)
- [ ] **内容已就绪吗？** (或是需要占位符)
- [ ] **询问过用户的偏好吗？** (颜色、风格、布局)

### 在交付前

- [ ] **看起来是否有高级感，而非普通模板？**
- [ ] **你会为这份设计感到自豪吗？**
- [ ] **它是否与你上一个项目有所区别？**

## 10. 复杂度预估 (Complexity Estimation)

### 快速项目 (小时级)

```
简单的落地页
小型个人作品集
基础表单
单个组件
```

→ 策略: 极速决策，专注执行。

### 中型项目 (天级)

```
多页面网站
带模块的仪表盘
电商分类页
复杂表单流
```

→ 策略: 建立 Token 系统，开发自定义组件。

### 大型项目 (周级)

```
完整的 SaaS 应用
大型电商平台
定制化设计系统
复杂业务工作流
```

→ 策略: 完整的全套设计系统、文档支撑、全面测试。

> **记住**: 这些模板展示的是 **结构 (STRUCTURE)** 和 **思维 (THINKING)** 流程。每一个项目都需要根据其独特的背景进行新鲜的颜色、排版和样式决策。**不清楚时尽管开口问。**
