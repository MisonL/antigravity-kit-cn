# 视觉特效参考 (Visual Effects Reference)

> 现代 CSS 特效原则与技术 —— 掌握概念，创造变化。
> **没有必须死板复制的固定数值 —— 请理解设计模式。**

---

## 1. 玻璃拟态原则 (Glassmorphism)

### 什么是成功的玻璃拟态

```
核心属性：
├── 半透明背景 (非纯色)
├── 背景模糊 (Backdrop blur, 磨砂玻璃效果)
├── 细腻的边框 (用于定义边缘)
└── 通常搭配：轻微的投影来增加深度
```

### 实现模式 (自定义数值)

```css
.glass {
    /* 透明度：根据内容可读性调整不透明度 */
    background: rgba(R, G, B, OPACITY);
    /* OPACITY: 暗色背景建议 0.1-0.3，亮色背景建议 0.5-0.8 */

    /* 模糊：数值越高，磨砂感越重 */
    backdrop-filter: blur(AMOUNT);
    /* AMOUNT: 8-12px 代表微妙，16-24px 代表强烈 */

    /* 边框：定义边缘，使其从背景中脱颖而出 */
    border: 1px solid rgba(255, 255, 255, OPACITY);
    /* OPACITY: 通常使用 0.1-0.3 */

    /* 圆角：契合你的设计系统 */
    border-radius: YOUR_RADIUS;
}
```

### 何时使用

- ✅ 覆盖在丰富色彩或图像背景上。
- ✅ 模态框、弹出层、交互卡片。
- ✅ 背景有滚动内容的导航栏。
- ❌ 文字密集的长篇内容 (会影响可读性)。
- ❌ 简单的纯色背景 (没有意义)。

## 2. 新拟态原则 (Neomorphism)

### 核心概念：利用“双阴影”模拟柔软的挤压效果

- **亮阴影**: 来自光源方向。
- **暗阴影**: 位于光源反方向。
- **背景**: 必须与父级背景完全一致 (同色)。

### 实现模式

```css
.neo-raised {
    /* 背景必须匹配父级 */
    background: SAME_AS_PARENT;

    /* 双阴影：光影方向 + 阴影方向 */
    box-shadow:
        OFFSET OFFSET BLUR rgba(light-color),
        -OFFSET -OFFSET BLUR rgba(dark-color);
}

.neo-pressed {
    /* 内部阴影创建“按下”效果 */
    box-shadow:
        inset OFFSET OFFSET BLUR rgba(dark-color),
        inset -OFFSET -OFFSET BLUR rgba(light-color);
}
```

## 3. 阴影层级原则 (Shadow Hierarchy)

### 概念：阴影指示海拔高度 (Elevation)

```
海拔越高 = 阴影越大且越柔和
├── 第 0 级: 无阴影 (平铺在表面)
├── 第 1 级: 微弱阴影 (轻微抬起)
├── 第 2 级: 中等阴影 (卡片, 按钮)
├── 第 3 级: 大阴影 (模态框, 下拉菜单)
└── 第 4 级: 深邃阴影 (悬浮元素)
```

### 自然阴影的原则

1. **Y 轴偏移大于 X 轴** (模拟光线从上方照射)。
2. **极低的不透明度** (5-15% 适合微妙感，15-25% 适合显著感)。
3. **多层投影** 增加真实感 (环境光 + 直射光)。

## 4. 渐变原则 (Gradients)

### 渐变类型

- **线性 (Linear)**: 沿直线变化。适用于背景、按钮、页眉。
- **径向 (Radial)**: 从中心向外扩散。适用于聚光灯效果、视觉焦点。
- **圆锥 (Conic)**: 绕中心点旋转变化。适用于饼图、创意环绕效果。

### 优秀的渐变规则

- 使用色环上 **相邻** 的颜色 (类比色)。
- 或者使用同一色相的不同明度。
- 避免使用互补色直接渐变 (会出现肮脏的灰色过渡)。
- 添加中间色控制点，使过渡更平滑。

---

## 5. 边框特效原则

### 渐变边框

通过伪元素配合渐变背景，利用 `mask` 或内容裁切来实现。

### 动画边框

使用旋转的锥形渐变 (conic-gradient) 结合裁切，制造跑马灯般的律动感。

---

## 6. 发光特效原则 (Glow)

### 文本发光 (Text Glow)

利用多层 `text-shadow` 叠加出光晕感。

### 元素发光 (Element Glow)

利用多层 `box-shadow` 叠加，并匹配元素主色以实现真实感。

---

## 7. 现代 CSS 技术预警

- **容器查询 (Container Queries)**: 组件响应其容器而非视口。
- **:has() 选择器**: 基于子级状态改变父级样式。
- **滚动驱动动画**: 动画进度与滚动位置一一对应。

---

## 9. 性能原则 (Performance)

### GPU 加速属性

- **transform**: 位移、缩放、旋转。
- **opacity**: 不透明度。
  通过这些属性实现动画，性能最优。

---

## 10. 特效选择检查清单

在应用特效前，请自检：

- [ ] **它服务于某个明确的目的吗？** (而非纯粹堆砌装饰)
- [ ] **它适合当前的背景吗？** (品牌、受众)
- [ ] **它是否与你之前的项目有所不同？** (避免设计同质化)
- [ ] **它是否可访问？** (对比度、动效敏感度)
- [ ] **它性能优异吗？** (尤其是在移动端)
- [ ] **询问过用户的审美偏好了吗？**

### 应避免的反模式 (Anti-Patterns)

- ❌ 每一个元素都套用玻璃拟态 (会显得很俗气)。
- ❌ 默认使用深色背景 + 霓虹色 (典型的偷懒 AI 审美)。
- ❌ **无深度的静态/扁平设计 (这是失败的，要有层级感)。**
- ❌ 伤害可读性的过度特效。
- ❌ 没有交互目的、单纯为了动而动的动画。

---

> **记住**: 特效是为了增强意义。请基于目的和背景来选择，而不是仅仅因为它“看起来很酷”。
