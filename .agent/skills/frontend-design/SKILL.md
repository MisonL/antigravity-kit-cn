## ⚠️ 关键：在假设之前先提问 (强制)

> **停下！如果用户的请求是开放式的，绝对不要直接套用你的偏好。**

### 当用户需求模糊时，请询问：

**未指定颜色？** 询问：

> “您偏好什么样的色盘？(蓝色/绿色/橙色/中性/其他？)”

**未指定风格？** 询问：

> “您想要追求什么样的风格？(极简/大胆/复古/未来感/有机感？)”

**未指定布局？** 询问：

> “您对布局有偏好吗？(单栏/网格/非对称/全宽？)”

### ⛔ 应避免的默认倾向 (反避风港行为)：

| AI 默认倾向                        | 为什么不好                       | 替代思考方式                         |
| ---------------------------------- | -------------------------------- | ------------------------------------ |
| **Bento 网格 (现代陈词滥调)**      | 几乎出现在所有 AI 设计中         | 为什么这些内容“需要”网格？           |
| **左右分割布局 (Hero Split)**      | 循规蹈矩且平庸                   | 试试巨型排版或纵向叙事？             |
| **弥散/极光渐变 (Mesh Gradients)** | “新”的偷懒背景方案               | 试试激进的颜色组合？                 |
| **玻璃拟态 (Glassmorphism)**       | AI 眼中的“高端”代名词            | 试试高对比度的实心扁平化？           |
| **深青色 / 金融蓝**                | 为了避开紫色禁令的“避风港”色     | 试试红色、黑色或霓虹绿？             |
| **“Orchestrate / Empower”**        | 典型的 AI 生成文案               | 人类会怎么表达这些？                 |
| 深色背景 + 霓虹发光                | 过度使用，一眼就能看出是 AI 作品 | 品牌真正需要的是什么？               |
| **圆角化一切**                     | 平庸且保守                       | 哪里可以尝试锋利的、粗野主义的边缘？ |

> 🔴 **“你选择的每一个‘安全’结构，都让你离平庸的模板更近一步。请勇敢冒险。”**

## 1. 约束分析 (始终放在第一步)

在进行任何设计工作前，先回答这些或询问用户：

| 约束条件   | 问题               | 为什么重要              |
| ---------- | ------------------ | ----------------------- |
| **时间线** | 有多少时间？       | 决定了设计的复杂度      |
| **内容**   | 已就绪还是占位符？ | 影响布局的灵活性        |
| **品牌**   | 是否有现成的指南？ | 可能直接规定了颜色/字体 |
| **技术栈** | 使用什么技术？     | 决定了实现的可能性      |
| **受众**   | 到底是谁？         | 驱动了所有的视觉决策    |

## 2. UX 心理学原则

### 核心定律 (请内化这些)

| 定律            | 原则                   | 应用                     |
| --------------- | ---------------------- | ------------------------ |
| **席克定律**    | 选项越多 = 决策越慢    | 限制选项，使用渐进式披露 |
| **菲茨定律**    | 越大 + 越近 = 越易点击 | 合理设置 CTA 按钮的大小  |
| **米勒定律**    | 工作记忆约为 7 个单项  | 将内容分组展示           |
| **冯·雷斯托夫** | 与众不同 = 令人难忘    | 赋予 CTA 视觉上的独特性  |
| **系列位置**    | 首尾信息最易被记住     | 将关键信息置于开头或结尾 |

## 3. 布局原则

### 黄金比例 (φ = 1.618)

用于比例和谐：内容与侧边栏比例约为 62% : 38%；各级标题字号建议按此比例递增。

### 8 点网格概念 (8-Point Grid)

所有的间距和尺寸都应是 8 的倍数，以确保设计的严谨性。

---

## 4. 色彩原则

### 60-30-10 法则

60% 主色/背景 (从容、中性)；30% 辅助色 (支撑区域)；10% 强调色 (CTA, 高亮, 注意力中心)。

详细色彩理论请参阅：[color-system.md](color-system.md)

---

## 5. 排版原则

### 比例尺选择

- 密集型 UI: 1.125-1.2
- 通用 Web: 1.25 (最常用)
- 品牌/Hero 区域: 1.5-1.618

详细排版原则请参阅：[typography-system.md](typography-system.md)

## 8. “Wow Factor” (惊艳感) 检查清单

- [ ] 大胆的留白 (奢侈感源于呼吸空间)。
- [ ] 微妙的深度与维度感。
- [ ] 平滑、有明确目的的动效。
- [ ] 对细节的极致关注 (对齐、一致性)。
- [ ] 连贯的视觉律动感。
- [ ] 定制化元素 (拒绝全盘使用默认样式)。

---

## 9. 反模式 (应避免的做法)

### ❌ 偷懒设计的表现

- 没有任何考虑就直接使用默认系统字体。
- 使用与各环节不匹配的素材图片。
- 间距混乱、不一致。
- 太多互不协调的颜色。
- 没有层级感的信息堆砌。
- 不符合无障碍标准的对比度。

### ❌ AI 典型倾向模式 (必须避开！)

- 每个项目都用同样的几种颜色。
- 默认深色 + 霓虹风格。
- **全场紫色 (紫色禁令 ✅)**。
- 即使是简单的落地页也套用 Bento 网格。
- 总是使用弥散渐变和发光特效。
- 不询问用户偏好就直接动手实现。

## 10. 决策流程总结

```
针对每一个设计任务：

1. 分析约束
   └── 时间、品牌、技术、受众分别是什么？
   └── 不清楚时 → 提问

2. 整理内容
   └── 现有什么内容？层级如何？

3. 设定风格方向
   └── 什么样的风格契合背景？
   └── 不清楚时 → 提问 (不要直接套用默认风格！)

4. 执行实现
   └── 应用上述原则，避开反模式

5. 评审反思
   └── “这真的服务于用户吗？”
   └── “这是否展现了我的最高水准？”
```
