## ⚠️ 关键：在假设前先提问 (强制要求)

> **停下！如果用户的请求是开放式的，千万不要直接套用你偏好的默认值。**

### 当用户需求模糊时，请提问：

**未指定颜色？** 询问：

> "您偏好哪种配色方案？(蓝色/绿色/橙色/中性色/其他？)"

**未指定风格？** 询问：

> "您追求的是哪种风格？(极简/大胆/复古/未来感/有机感？)"

**未指定布局？** 询问：

> "您对布局有偏好吗？(单列/网格/不对称/全宽布局？)"

### ⛔ 应避免的 AI 默认倾向 (反避风港原则)：

| AI 默认倾向                   | 为什么不好               | 换个思路                           |
| ----------------------------- | ------------------------ | ---------------------------------- |
| **Bento 网格 (现代陈词滥调)** | 几乎所有的 AI 设计都在用 | 为什么这段内容“必须”用网格？       |
| **左右分割布局 (Hero Split)** | 循规蹈矩且枯燥           | 试试巨型排版或纵向叙事方案？       |
| **弥散渐变 (Mesh Gradients)** | “新型”偷懒背景           | 试试大胆激进的配色组合？           |
| **玻璃拟态 (Glassmorphism)**  | AI 眼中的“高端”          | 试试实心、高对比度的扁平设计？     |
| **深青色 / 金融蓝**           | 躲避紫色禁令的避风港     | 为什么不用红色、黑色或荧光绿？     |
| **"赋能 / 协同" 等词汇**      | 典型的 AI 生成文案       | 真人会怎么说这些内容？             |
| **黑底 + 霓虹发光**           | 过度使用，一股 AI 味     | 该“品牌”真正需要的是什么？         |
| **这种地方都用圆角**          | 庸常且保险               | 哪里可以用犀利、包豪斯风格的棱角？ |

> 🔴 **"你选择的每一个‘安全’结构，都让你离平庸的模板更近一步。请去冒险。"**

## 1. 约束分析 (永远排在第一步)

在进行任何设计工作前，先回答这些问题或询问用户：

| 约束条件     | 问题                 | 为什么这很重要      |
| ------------ | -------------------- | ------------------- |
| **时间线**   | 有多少开发时间？     | 决定设计的复杂上限  |
| **内容**     | 已准备好还是占位符？ | 影响布局的灵活性    |
| **品牌**     | 是否有现有指南？     | 可能限制了颜色/字体 |
| **技术栈**   | 使用什么框架？       | 影响交互实现的能力  |
| **目标受众** | 具体是谁？           | 驱使所有的视觉决策  |

### 受众 → 设计策略

| 受众               | 考虑因素                     |
| ------------------ | ---------------------------- |
| **Z 世代 (Gen Z)** | 大胆、快速、移动优先、真实感 |
| **千禧一代**       | 整洁、极简、价值驱动         |
| **X 世代 (Gen X)** | 熟悉感、值得信赖、目标明确   |
| **婴儿潮一代**     | 易读性、高对比度、操作简单   |
| **B2B 领域**       | 专业感、数据导向、信任感     |
| **奢侈品牌**       | 克制的优雅、大量的留白       |

## 2. UX 心理学原则

### 核心定律 (必须内化)

| 定律                        | 原则                   | 应用                          |
| --------------------------- | ---------------------- | ----------------------------- |
| **席克定律 (Hick's Law)**   | 选项越多 = 决策越慢    | 限制选项，使用渐进式披露      |
| **菲茨定律 (Fitts' Law)**   | 更大 + 更近 = 更易点击 | 合理设置 CTA 按钮的大小和位置 |
| **米勒定律 (Miller's Law)** | 短时记忆只能记住 ~7 项 | 将内容分块 (Chunking) 处理    |
| **冯·雷斯托夫效应**         | 差异化 = 易记          | 让 CTA 在视觉上具备显著差异   |
| **系列位置效应**            | 开头和结尾最易记住     | 关键信息放在最前或最后        |

### 情感设计层级

```
本能层 (Visceral) → 第一印象：颜色、图像、整体氛围
行为层 (Behavioral) → 使用体验：速度、反馈、效率
反思层 (Reflective) → 使用之后：“我喜欢这个产品带来的自我认同”
```

### 建立信任感

- 针对敏感操作提供安全指示。
- 在合适位置展示社交证明 (Social proof)。
- 提供清晰的联系方式/支持入口。
- 保持一致且专业的视觉设计。
- 政策透明化。

## 3. 布局原则

### 黄金分割 (φ = 1.618)

```
用于比例和谐：
├── 主内容 : 侧边栏 = 约 62% : 38%
├── 各级标题大小 = 上级标题 ÷ 1.618 (追求戏剧化的比例)
├── 间距序列: 小 → 中 → 大 (每一级 × 1.618)
```

### 8 像素网格概念

```
所有的间距和尺寸均为 8 的倍数：
├── 极细微: 4px (微调使用的半步)
├── 小: 8px
├── 中: 16px
├── 大: 24px, 32px
├── 特大: 48px, 64px, 80px
└── 根据内容密度进行灵活调整
```

### 关键尺寸原则

| 元素         | 考虑因素                          |
| ------------ | --------------------------------- |
| **触摸目标** | 最小舒适点击尺寸 (通常为 44-48px) |
| **按钮**     | 高度应基于重要性层级区分          |
| **输入框**   | 高度应与按钮匹配以保持对齐        |
| **卡片**     | 边距一致，保持“呼吸感”            |
| **阅读宽度** | 最佳每行字符数为 45-75 个字符     |

## 4. 颜色原则

### 60-30-10 法则

```
60% → 主色/背景色 (冷静、中性的基调)
30% → 辅助色 (支撑区域)
10% → 强调色 (CTA, 高亮, 全场焦点)
```

### 颜色心理学 (辅助决策)

| 如果你需要... | 考虑色调             | 避开             |
| ------------- | -------------------- | ---------------- |
| 信任, 冷静    | 蓝色系               | 充满攻击性的红色 |
| 成长, 自然    | 绿色系               | 冰冷的工业灰     |
| 能量, 紧迫    | 橙色, 红色           | 消极的蓝色       |
| 奢华, 创意    | 深青色, 金色, 翡翠绿 | 廉价的高亮色     |
| 整洁, 极简    | 中性色               | 压倒性的色彩堆砌 |

### 颜色选择流程

1. **行业背景是什么？** (缩小范围)
2. **想传达什么情绪？** (挑选主色)
3. **亮色还是暗色模式？** (确定基调)
4. **如果未指定，请询问用户。**

详细色彩理论参考：[color-system.md](color-system.md)

## 5. 排版原则 (Typography)

### 比例尺选择

| 内容类型      | 缩放比例  | 感受          |
| ------------- | --------- | ------------- |
| 密集 UI       | 1.125-1.2 | 紧凑, 高效    |
| 通用网页      | 1.25      | 均衡 (最常用) |
| 社论/博客     | 1.333     | 易读, 宽裕    |
| Hero/大屏展示 | 1.5-1.618 | 极具冲击力    |

### 搭配理念

```
对比 + 和谐：
├── 差异足够大，以显现层级 (Hierarchy)
├── 相似点足够多，以保持凝聚力 (Cohesion)
└── 通常：展示型字体 + 中性字体，或 衬线体 + 无衬线体
```

### 易读性规则

- **行长**: 最佳 45-75 个字符。
- **行高**: 正文建议 1.4-1.6。
- **对比度**: 检查 WCAG 无障碍要求。
- **字号**: Web 端正文建议 16px+。

详细排版参考：[typography-system.md](typography-system.md)

## 6. 视觉效果原则

### 玻璃拟态 (在合适时使用)

```
核心属性：
├── 半透明背景
├── 背景模糊 (Backdrop blur)
├── 微细的边框以定义轮廓
└── ⚠️ **警告：** 标准的蓝白玻璃拟态已沦为陈词滥调。要么用得激进一点，要么不用。
```

### 阴影层级 (Hierarchy)

```
高度感概念：
├── 元素越高 = 阴影越大/越模糊
├── Y 轴偏移 > X 轴偏移 (光源从上方照射)
├── 多层组合 = 更加逼真
└── 暗色模式：可能需要发光 (Glow) 代替阴影
```

### 渐变使用

```
和谐的渐变：
├── 色环上的邻近色 (类比色)
├── 或者 相同色调，不同明度
├── 避开突兀的互补色对
├── 🚫 **禁止使用弥散渐变 (Mesh/Aurora Gradients)** (指那种乱飘的色块)
└── 确保每个项目之间的风格跨度足够大
```

详细效果指南参考：[visual-effects.md](visual-effects.md)

## 7. 动画原则

### 节奏概念 (Timing)

```
时长基于：
├── 距离 (越远越长)
├── 尺寸 (越大越慢)
├── 重要性 (关键动作必须清晰)
└── 背景 (紧迫动作快，奢华感动作慢)
```

### 缓动函数选择 (Easing)

| 动作   | 缓动方式    | 原因             |
| ------ | ----------- | ---------------- |
| 进入   | Ease-out    | 减速，平稳停靠   |
| 离开   | Ease-in     | 加速，干脆利落   |
| 强调   | Ease-in-out | 丝滑，从容       |
| 趣味性 | Bounce      | 有趣，充满生命力 |

### 性能考虑

- 仅对 `transform` 和 `opacity` 进行动画处理。
- 尊重用户的 `reduced-motion` (减少动画) 偏好。
- 在低端设备上进行测试。

动画模式参考：[animation-guide.md](animation-guide.md)，进阶参考：[motion-graphics.md](motion-graphics.md)

## 8. "Wow Factor" 检查清单

### 高级感指标 (Premium Indicators)

- [ ] 慷慨的留白 (奢华 = 呼吸空间)。
- [ ] 微妙的深度和维度感。
- [ ] 丝滑且有目的性的动画。
- [ ] 对细节的极致关注 (对齐、一致性)。
- [ ] 凝聚的视觉韵律。
- [ ] 定制化元素 (不全用默认值)。

### 信任感建立

- [ ] 在恰当位置展示安全提示。
- [ ] 社交证明 / 客户评价。
- [ ] 清晰的价值主张 (Value proposition)。
- [ ] 专业的图像/摄影作品。
- [ ] 一致的设计语言。

### 情感触发点

- [ ] 能够激发预期情感的 Hero 区域。
- [ ] 人文元素 (人脸、故事)。
- [ ] 进度/成就感指示器。
- [ ] 令人愉悦的小细节 (Moments of delight)。

## 9. 反模式 (应避免的做法)

### ❌ 偷懒设计的表现

- 未经思考就直接用系统默认字体。
- 使用不匹配的图库素材。
- 间距不一致。
- 颜色过多且相互冲突。
- 只有大段文字而无排版层级。
- 无法满足无障碍要求的对比度。

### ❌ AI 惯性模式 (必须避开！)

- **每个项目都用同样的几种颜色。**
- **默认就是黑底加霓虹光。**
- **到处都是紫色/紫罗兰色 (紫色禁令 ✅)。**
- **连最简单的 Landing Page 都套用 Bento 网格。**
- **滥用弥散渐变和发光特效。**
- **一成不变的布局结构 / 像 Vercel 官网的克隆版。**
- **从不询问用户的偏好。**

### ❌ 暗黑模式 (不道德)

- 隐藏额外费用。
- 制造虚假的紧迫感。
- 诱导性操作。
- 误导性 UI。
- 确认羞辱 (Confirmshaming)。

## 10. 决策流程总结

```
针对每一个设计任务：

1. 约束分析 (CONSTRAINTS)
   └── 时间线、品牌、技术、受众分别是什么？
   └── 不明确？去提问！

2. 内容梳理 (CONTENT)
   └── 现有什么内容？
   └── 信息的优先级层级是什么？

3. 风格定位 (STYLE DIRECTION)
   └── 什么风格最契合当前背景？
   └── 不明确？去提问！(不要套默认值)

4. 执行实现 (EXECUTION)
   └── 应用上述原则
   └── 检查是否落入反模式

5. 评审回顾 (REVIEW)
   └── “这真的服务于用户吗？”
   └── “这是否脱离了我的 AI 默认模板？”
   └── “我会为这个设计感到自豪吗？”
```
