## ⚠️ 关键：假设前先询问 (强制)

> **停止！如果用户的请求是开放式的，不要默认为你的喜好。**

### 当用户提示模糊时，询问：

**未指定颜色？** 问：

> "您喜欢什么色调？(蓝色/绿色/橙色/中性/其他？)"

**未指定风格？** 问：

> "您想要什么风格？(极简/大胆/复古/未来主义/有机？)"

**未指定布局？** 问：

> "您有布局偏好吗？(单列/网格/不对称/全宽？)"

### ⛔ 避免的 AI 默认倾向 (反避风港):

| AI 默认倾向                               | 为什么不好         | 应该思考                         |
| ----------------------------------------- | ------------------ | -------------------------------- |
| **Bento Grids (便当盒网格)**              | 每个 AI 设计都在用 | 为什么这个内容 **需要** 网格？   |
| **Hero 左右分割**                         | 可预测且无聊       | 巨大的排版或垂直叙事如何？       |
| **Mesh/Aurora 渐变**                      | "新"的懒惰背景     | 什么是激进的颜色配对？           |
| **玻璃拟态 (Glassmorphism)**              | AI 眼中的"高级"    | 扎实、高对比度的扁平化如何？     |
| **深青色 / 金融蓝**                       | 紫色禁令后的避风港 | 为什么不是红色、黑色或霓虹绿？   |
| **"编排 / 赋能" (Orchestrate / Empower)** | AI 生成的文案      | 人类会怎么说？                   |
| 暗色背景 + 霓虹光                         | 滥用，"AI 风格"    | **品牌**实际上需要什么？         |
| **万物圆角**                              | 通用/安全          | 哪里可以使用锐利、野兽派的边缘？ |

> 🔴 **"你选择的每一个'安全'结构都让你离通用模板更近一步。要冒险。"**

## 1. 约束分析 (总是第一步)

在任何设计工作之前，回答这些问题或询问用户：

| 约束       | 问题           | 为什么重要        |
| ---------- | -------------- | ----------------- |
| **时间线** | 多少时间？     | 决定复杂性        |
| **内容**   | 就绪还是占位？ | 影响布局灵活性    |
| **品牌**   | 现有指南？     | 可能规定颜色/字体 |
| **技术**   | 什​​么技术栈？ | 影响能力          |
| **受众**   | 具体是谁？     | 驱动所有视觉决策  |

### 受众 → 设计方法

| 受众           | 思考方向                   |
| -------------- | -------------------------- |
| **Z 世代**     | 大胆，快速，移动优先，真实 |
| **千禧一代**   | 干净，极简，价值驱动       |
| **X 世代**     | 熟悉，值得信赖，清晰       |
| **婴儿潮一代** | 易读，高对比度，简单       |
| **ToB (B2B)**  | 专业，数据为中心，信任     |
| **奢侈品**     | 克制的优雅，留白           |

## 2. UX 心理学原则 (UX Psychology Principles)

### 核心定律 (内化这些)

| 定律                        | 原则                     | 应用                     |
| --------------------------- | ------------------------ | ------------------------ |
| **希克定律 (Hick's Law)**   | 更多选择 = 更慢决策      | 限制选项，使用渐进式披露 |
| **费茨定律 (Fitts' Law)**   | 更大 + 更近 = 更容易点击 | 适当调整 CTA 大小        |
| **米勒定律 (Miller's Law)** | 工作记忆约 7 项          | 将内容分组 (Chunking)    |
| **冯·雷斯托夫效应**         | 不同 = 难忘              | 让 CTA 在视觉上独特      |
| **序列位置效应**            | 最先/最后记得最牢        | 关键信息在开始/结束      |

### 情感设计层次

```
本能层 (VISCERAL) (即时)  → 第一印象：颜色，图像，整体感觉
行为层 (BEHAVIORAL) (使用)    → 使用中：速度，反馈，效率
反思层 (REFLECTIVE) (记忆) → 之后："我喜欢这东西所代表的我"
```

### 建立信任

- 敏感操作上的安全指示
- 相关处的社会证明
- 清晰的联系/支持入口
- 一致、专业的设计
- 透明的政策

## 3. 布局原则 (Layout Principles)

### 黄金比例 (φ = 1.618)

```
用于比例和谐：
├── 内容 : 侧边栏 = 大约 62% : 38%
├── 每个标题大小 = 上一级 × 1.618 (用于戏剧性比例)
├── 间距可以遵循：sm → md → lg (每个 × 1.618)
```

### 8 点网格概念

```
所有间距和尺寸均为 8 的倍数：
├── 紧凑: 4px (微调的半步)
├── 小: 8px
├── 中: 16px
├── 大: 24px, 32px
├── 特大: 48px, 64px, 80px
└── 根据内容密度调整
```

### 关键尺寸原则

| 元素         | 考虑                 |
| ------------ | -------------------- |
| **触摸目标** | 最小舒适点击尺寸     |
| **按钮**     | 基于重要性层级的高度 |
| **输入框**   | 匹配按钮高度以对齐   |
| **卡片**     | 一致的内边距，透气   |
| **阅读宽度** | 45-75 字符为最佳     |

## 4. 颜色原则 (Color Principles)

### 60-30-10 规则

```
60% → 主色/背景 (平静，中性基底)
30% → 辅助色 (支撑区域)
10% → 强调色 (CTA，高亮，注意力)
```

### 色彩心理学 (用于决策)

| 如果你需要... | 考虑色相             | 避免         |
| ------------- | -------------------- | ------------ |
| 信任，平静    | 蓝色系               | 激进的红色   |
| 成长，自然    | 绿色系               | 工业灰色     |
| 能量，紧迫    | 橙色，红色           | 被动的蓝色   |
| 奢华，创造力  | 深青色，金色，祖母绿 | 廉价感的亮色 |
| 干净，极简    | 中性色               | 压倒性的颜色 |

### 选择流程

1. **行业是什么？** (缩小选项)
2. **情感是什么？** (挑选主色)
3. **亮色还是暗色模式？** (设定基调)
4. **询问用户** 如未指定

关于详细色彩理论：[color-system.md](color-system.md)

## 5. 排版原则 (Typography Principles)

### 比例选择

| 内容类型  | 比例系数  | 感觉          |
| --------- | --------- | ------------- |
| 密集 UI   | 1.125-1.2 | 紧凑，高效    |
| 通用 Web  | 1.25      | 平衡 (最常见) |
| 编辑/阅读 | 1.333     | 易读，宽敞    |
| Hero/展示 | 1.5-1.618 | 戏剧性冲击    |

### 配对概念

```
对比 + 和谐:
├── 足够不同以区分层级
├── 足够相似以保持凝聚力
└── 通常：展示字体 + 中性字体，或衬线 + 无衬线
```

### 易读性规则

- **行长**: 45-75 字符最佳
- **行高**: 正文 1.4-1.6
- **对比度**: 检查 WCAG 要求
- **字号**: Web 正文 16px+

关于详细排版：[typography-system.md](typography-system.md)

## 6. 视觉效果原则 (Visual Effects Principles)

### 玻璃拟态 (适当时使用)

```
关键属性:
├── 半透明背景
├── 背景模糊 (Backdrop blur)
├── 微妙的边框用于定义
└── ⚠️ **警告:** 标准的蓝/白玻璃拟态是现代陈词滥调。要么激进地用，要么别用。
```

### 阴影层级

```
海拔概念:
├── 更高的元素 = 更大的阴影
├── Y 偏移 > X 偏移 (光线来自上方)
├── 多层阴影 = 更真实
└── 暗色模式：可能需要发光代替
```

### 渐变使用

```
和谐渐变:
├── 色轮上的相邻颜色 (类比色)
├── 或相同色相，不同亮度
├── 避免刺眼的互补色对
├── 🚫 **无 Mesh/Aurora 渐变** (漂浮的斑点)
└── 在不同项目中激进地变化
```

关于完整效果指南：[visual-effects.md](visual-effects.md)

## 7. 动画原则 (Animation Principles)

### 时序概念

```
持续时间基于：
├── 距离 (更远 = 更长)
├── 大小 (更大 = 更慢)
├── 重要性 (关键 = 清晰)
└── 上下文 (紧急 = 快，奢华 = 慢)
```

### 缓动选择 (Easing Selection)

| 动作 | 缓动        | 原因           |
| ---- | ----------- | -------------- |
| 进入 | Ease-out    | 减速，安顿下来 |
| 离开 | Ease-in     | 加速，退出     |
| 强调 | Ease-in-out | 平滑，刻意     |
| 俏皮 | Bounce      | 有趣，充满活力 |

### 性能

- 仅对 transform 和 opacity 进行动画
- 尊重减弱动态 (reduced-motion) 偏好
- 在低端设备上测试

关于动画模式：[animation-guide.md](animation-guide.md)，进阶：[motion-graphics.md](motion-graphics.md)

## 9. 反模式 (不要做的事)

### ❌ 懒惰设计指标

- 未经考虑的默认系统字体
- 不匹配的库存图片
- 不一致的间距
- 太多相互竞争的颜色
- 没有层级的文字墙
- 不可访问的对比度

### ❌ AI 倾向模式 (避免！)

- **每个项目都用相同的颜色**
- **默认使用暗色 + 霓虹**
- **到处都是紫色/紫罗兰色 (紫色禁令 ✅)**
- **简单的落地页使用 Bento Grids**
- **Mesh 渐变和发光效果**
- **相同的布局结构 / Vercel 克隆**
- **不询问用户偏好**

### ❌ 暗黑模式 (不道德)

- 隐藏成本
- 虚假紧迫感
- 强制行动
- 欺骗性 UI
- 确认羞辱 (Confirmshaming)

## 10. 决策过程总结

```
对于每一个设计任务：

1. 约束 (CONSTRAINTS)
   └── 时间线、品牌、技术、受众是什么？
   └── 如果不清楚 → 询问

2. 内容 (CONTENT)
   └── 存在什么内容？
   └── 层级是什么？

3. 风格方向 (STYLE DIRECTION)
   └── 什​​么适合上下文？
   └── 如果不清楚 → 询问 (不要默认！)

4. 执行 (EXECUTION)
   └── 应用上述原则
   └── 检查反模式

5. 审查 (REVIEW)
   └── "这服务于用户吗？"
   └── "这与我的默认设置不同吗？"
   └── "我会为此感到自豪吗？"
```
