# 配色系统参考

> 色彩理论、选择流程与决策指南。
> **不背固定十六进制值——学会思考颜色。**

---

## 1. 色彩理论基础

### 色相环

```
                    黄色
                      │
           黄绿       │       黄橙
              ╲       │       ╱
               ╲      │      ╱
    绿色 ─────────── ● ─────────── 橙色
               ╱      │      ╲
              ╱       │       ╲
           蓝绿       │       红橙
                      │
                     红色
                      │
                     紫色
                  ╱       ╲
             蓝紫         红紫
                  ╲       ╱
                    蓝色
```

### 色彩关系

| 方案 | 如何创建 | 何时使用 |
|------|---------|---------|
| **单色（Monochromatic）** | 选择一个色相，仅变化明度/饱和度 | 极简、专业、一致 |
| **相邻色（Analogous）** | 选择色轮相邻的 2-3 个色相 | 和谐、平静、自然感 |
| **互补色（Complementary）** | 选择色轮相对的色相 | 高对比、醒目、吸引注意 |
| **分裂互补（Split-Complementary）** | 主色 + 互补色相邻两色 | 动态但平衡 |
| **三分色（Triadic）** | 色轮等距三色 | 活泼、创造力强 |

### 如何选择方案：
1. **项目气质是什么？** 冷静 → 相邻色；大胆 → 互补色。
2. **需要多少颜色？** 极简 → 单色；复杂 → 三分色。
3. **受众是谁？** 保守 → 单色；年轻 → 三分色。

## 2. 60-30-10 规则

### 分布原则
```
┌─────────────────────────────────────────────────┐
│                                                 │
│     60% 主色（背景、大面积区域）                │
│     → 应偏中性或平静                            │
│     → 决定整体基调                              │
│                                                 │
├────────────────────────────────────┬────────────┤
│                                    │            │
│   30% 次级色                       │ 10% 点缀色 │
│   （卡片、区块、标题）             │ （CTA（行动号召）、    │
│   → 支撑而不抢戏                   │ 高亮）    │
│                                    │ → 吸引注意 │
└────────────────────────────────────┴────────────┘
```

### 实现方式
```css
:root {
  /* 60% - 依据浅/深色模式与情绪选择 */
  --color-bg: /* 中性色：白、米白或深灰 */
  --color-surface: /* 比背景略有差异 */

  /* 30% - 依据品牌或场景选择 */
  --color-secondary: /* 主色的柔和版本或中性色 */

  /* 10% - 依据动作/情绪选择 */
  --color-accent: /* 鲜明、吸引注意 */
}
```

## 3. 色彩心理 - 含义与选择

### 按场景选择

| 项目类型 | 参考色系 | 原因 |
|---------|---------|------|
| **金融、科技、医疗** | 蓝、青 | 信任、稳定、平静 |
| **环保、健康、自然** | 绿、土色 | 成长、健康、有机 |
| **食品、能量、年轻** | 橙、黄、暖色 | 食欲、兴奋、温暖 |
| **奢华、美妆、创意** | 深青、金、黑 | 高级、精致 |
| **紧迫、促销、警报** | 红、橙 | 行动、注意、热度 |

### 情绪联想（用于决策）

| 色系 | 正向联想 | 注意事项 |
|------|---------|---------|
| **蓝** | 信任、平静、专业 | 可能显得冷、过度企业化 |
| **绿** | 成长、自然、成功 | 过度使用会显得无聊 |
| **红** | 热情、紧迫、能量 | 刺激强，谨慎使用 |
| **橙** | 温暖、友好、创意 | 过饱和会显得廉价 |
| **紫** | ⚠️ **禁用** - AI 过度使用！ | 改用深青/酒红/祖母绿 |
| **黄** | 乐观、注意、快乐 | 可读性差，建议作点缀 |
| **黑** | 优雅、力量、现代 | 可能显得沉重 |
| **白** | 干净、极简、开放 | 可能显得冷淡 |

### 选择流程：
1. **行业是什么？** → 缩小到 2-3 个色系
2. **情绪是什么？** → 选择主色
3. **对比如何？** → 决定浅/深色模式
4. **询问用户** → 未明确前先确认

## 4. 调色板生成原则

### 从单色出发（HSL 方法）

不要背十六进制，学会**操作 HSL**：

```
HSL = Hue（色相）, Saturation（饱和度）, Lightness（明度）

Hue（0-360）：色相
  0/360 = 红
  60 = 黄
  120 = 绿
  180 = 青
  240 = 蓝
  300 = 紫

Saturation（0-100%）：饱和度
  低 = 柔和、克制
  高 = 鲜明、活力

Lightness（0-100%）：明度
  0% = 黑
  50% = 纯色
  100% = 白
```

### 生成完整色阶

给定任意基色，生成等级：

```
Lightness Scale:
  50  （最浅） → L: 97%
  100           → L: 94%
  200           → L: 86%
  300           → L: 74%
  400           → L: 66%
  500（基色）   → L: 50-60%
  600           → L: 48%
  700           → L: 38%
  800           → L: 30%
  900（最深）   → L: 20%
```

### 饱和度调整

| 场景 | 饱和度水平 |
|------|-----------|
| **专业/企业** | 较低（40-60%） |
| **活泼/年轻** | 较高（70-90%） |
| **深色模式** | 降低 10-20% |
| **无障碍** | 保证对比度，必要时调整 |

## 5. 基于上下文的选择指南

### 不要直接抄配色，按流程选择：

**步骤 1：识别上下文**
```
项目类型？
├── 电商 → 需要信任 + 紧迫的平衡
├── SaaS（软件即服务）/仪表盘 → 低疲劳、数据导向
├── 健康/医疗 → 平静、自然
├── 奢华/高端 → 克制优雅
├── 创意/作品集 → 具个性、易记忆
└── 其他 → 询问用户
```

**步骤 2：选择主色系**
```
基于上下文，选一个：
- 蓝色系（信任）
- 绿色系（成长）
- 暖色系（能量）
- 中性色系（优雅）
- 或直接询问用户偏好
```

**步骤 3：确定浅色/深色模式**
```
考虑：
- 用户偏好？
- 行业习惯？
- 内容类型？（文字多 → 浅色优先）
- 使用时间？（夜间应用 → 深色优先）
```

**步骤 4：按原则生成色板**
- 使用 HSL 调整
- 遵循 60-30-10 规则
- 检查对比度（WCAG，无障碍标准）
- 用真实内容测试

## 6. 深色模式原则

### 关键规则（不背固定值）

1. **不要纯黑** → 使用略带色相的深灰
2. **不要纯白文字** → 使用 87-92% 明度
3. **降低饱和度** → 深色模式中色彩更刺眼
4. **层级 = 亮度** → 更高层级略亮

### 深色模式对比

```
背景层级（从暗到亮）：
Layer 0（基底）    → 最暗
Layer 1（卡片）    → 稍亮
Layer 2（弹窗）    → 更亮
Layer 3（浮层）    → 最亮的深色
```

### 深色模式色彩调整

| 浅色模式 | 深色模式调整 |
|----------|---------------|
| 高饱和点缀 | 降低饱和度 10-20% |
| 纯白背景 | 略带品牌色的深灰 |
| 黑色文字 | 浅灰（非纯白） |
| 彩色背景 | 降饱和、更深版本 |

## 7. 无障碍指南

### 对比度要求（WCAG，无障碍标准）

| 等级 | 正文 | 大文字 |
|------|------|--------|
| AA（最低） | 4.5:1 | 3:1 |
| AAA（增强） | 7:1 | 4.5:1 |

### 如何检查对比度

1. **计算亮度**
2. **计算比值**： (亮色 + 0.05) / (暗色 + 0.05)
3. **调整直至达标**

### 安全模式

| 用例 | 指南 |
|------|------|
| **浅色背景文字** | 明度 ≤ 35% |
| **深色背景文字** | 明度 ≥ 85% |
| **白底主色** | 确保足够深 |
| **按钮** | 背景与文字高对比 |

## 9. 反模式（避免）

### ❌ 不要：
- 每个项目都用同一套色值
- 默认紫色/紫罗兰（AI 倾向）
- 默认深色 + 霓虹（AI 倾向）
- 纯黑（#000000）背景
- 深色背景上用纯白文字（#FFFFFF）
- 忽略行业上下文
- 不询问用户偏好

### ✅ 要做：
- 每个项目生成新的色板
- 询问用户配色偏好
- 考虑行业与受众
- 用 HSL 灵活调整
- 测试对比度与无障碍
- 提供浅色与深色方案

---

> **记住：** 颜色是决策，不是默认值。每个项目都应基于其独特上下文做有意识的选择。
