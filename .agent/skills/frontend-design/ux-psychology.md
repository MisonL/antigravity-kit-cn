# UX 心理学参考 (UX Psychology Reference)

> 深度探讨 UX 定律、情感化设计、信任建立以及行为心理学。

---

## 1. 核心 UX 定律 (Core UX Laws)

### 席克定律 (Hick's Law)

**原则：** 做出决定所需的时间随着选项数量的增加而呈对数增长。

```
决策时间 = a + b × log₂(n + 1)
其中 n = 选项数量
```

**应用：**

- 导航：顶级项目上限为 5-7 个。
- 表单：拆分为多个步骤 (渐进式披露)。
- 选项：尽可能提供默认选择。
- 过滤器：优先展示最常用的，隐藏高级选项。

**示例：**

```
❌ 错误：一个导航中有 15 个菜单项
✅ 正确：5 个主要类别 + "更多"

❌ 错误：一次显示 20 个表单字段
✅ 正确：分为 3 步的向导，每步 5-7 个字段
```

---

### 菲茨定律 (Fitts' Law)

**原则：** 到达目标的时间是距离和目标大小的函数。

```
MT = a + b × log₂(1 + D/W)
其中 D = 距离，W = 宽度
```

**应用：**

- CTAs: 放大主按钮 (高度建议至少 48px)。
- 触摸目标: 移动端最小 44×44px。
- 布局: 将重要操作放置在鼠标自然停留的位置。
- 边界: "魔法边界" (无限边缘 = 易于点击)。

**按钮尺寸：**

```css
/* 按重要性排序 */
.btn-primary {
    height: 48px;
    padding: 0 24px;
}
.btn-secondary {
    height: 40px;
    padding: 0 16px;
}
.btn-tertiary {
    height: 36px;
    padding: 0 12px;
}

/* 移动端触摸目标 */
@media (hover: none) {
    .btn {
        min-height: 44px;
        min-width: 44px;
    }
}
```

---

### 米勒定律 (Miller's Law)

**原则：** 普通人的工作记忆平均能容纳 7±2 个信息块 (Chunks)。

**应用：**

- 列表: 将项目分组，每组 5-7 个。
- 导航: 菜单项上限为 7 个。
- 内容: 使用标题拆分长篇内容。
- 电话号码: 使用分节符 (如 555-123-4567)。

**分块示例：**

```
❌ 5551234567
✅ 555-123-4567

❌ 没有分隔的长文本段落
✅ 短段落
   配合项目符号
   以及子标题
```

---

### 冯·雷斯托夫效应 (孤立效应 / Von Restorff Effect)

**原则：** 最显眼的那个项目最容易被记住。

**应用：**

- CTA 按钮: 使用与众不同的主色。
- 定价: 高亮推荐方案。
- 重要信息: 视觉上的差异化处理。
- 新功能: 使用角标 (Badge) 或气泡提醒。

**示例：**

```css
/* 所有按钮为灰色，主按钮突出显示 */
.btn {
    background: #e5e7eb;
}
.btn-primary {
    background: #3b82f6;
}

/* 高亮推荐方案 */
.pricing-card {
    border: 1px solid #e5e7eb;
}
.pricing-card.popular {
    border: 2px solid #3b82f6;
    box-shadow: var(--shadow-lg);
}
```

---

### 系列位置效应 (Serial Position Effect)

**原则：** 列表开头 (首因效应) 和结尾 (近因效应) 的项目最容易被记住。

**应用：**

- 导航: 将最重要的项目放在首尾。
- 列表: 关键信息放在顶部和底部。
- 表单: 最关键的字段放在开始位置。
- CTAs: 在长页面的顶部和底部重复。

**示例：**

```
导航栏: 首页 | [中间项] | 联系我们

长落地页:
- Hero 区域的 CTA (顶部)
- 内容区块
- 底部重复 CTA
```

---

### 雅各布定律 (Jakob’s Law)

**原则：** 用户大部分时间都在别人的网站上。他们希望你的网站工作方式与他们已经熟悉的网站相同。

**应用：**

- **模式:** 搜索框、购物车等放在标准位置。
- **心理模型:** 使用熟悉的图标 (如放大镜代表搜索)。
- **词汇:** 使用“登录”而不是“进入传送门”。
- **布局:** Logo 放在左上角以便返回“首页”。
- **轴向交互:** 向右滑动返回/下一步应感觉像原生操作。
- **反馈:** 标准配色 (红=错误，绿=成功)。

**示例：**

```
❌ 错误：点击网站 Logo 跳转到“关于我们”页面。
✅ 正确：点击 Logo 始终返回首页。

❌ 错误：使用“星号”图标代表“删除”。
✅ 正确：使用“垃圾桶”图标代表“删除”。
```

---

### 泰斯勒定律 (复杂度守恒定律 / Tesler’s Law)

**原则：** 每个系统都有一定的复杂度是无法减少的，只能从用户端转移到软件端。

**应用：**

- **后端处理:** 系统自动处理格式 (如货币符号、手机号空格)。
- **智能检测:** 根据邮编自动识别城市。
- **自动化:** 预填已登录用户的数据。
- **个性化:** 根据之前的回答只显示相关的字段。
- **默认值:** 常用设置的智能默认值。
- **集成:** 使用 SSO (社交登录) 来减轻注册负担。

**示例：**

```
❌ 错误：要求用户在表单的每个价格字段前输入“USD $”。
✅ 正确：应用根据用户位置自动添加“$”前缀。

❌ 错误：强制用户手动选择“卡种”(Visa/Mastercard)。
✅ 正确：根据输入的前四位数字自动检测。
```

---

### 帕金森定律 (Parkinson’s Law)

**原则：** 只要还有时间，工作就会不断膨胀，直到用完所有的时间。

**应用：**

- **效率:** 使用“自动保存”减少任务完成时间。
- **速度:** 限制转化漏斗中的步骤。
- **清晰度:** 使用清晰的标签防止用户为寻找含义而进行的“悬停探索”。
- **反馈:** 实时验证以阻止用户在错误上浪费时间。
- **引导:** 为高级用户提供快速的“快速设置”。
- **约束:** 在输入框中设置字符限制以集中思绪。

**示例：**

```
❌ 错误：一个 10 页的注册表单，允许用户跳转并丢失数据。
✅ 正确：使用 Google 或 Apple ID 进行“一键登录”。

❌ 错误：给用户无限的时间来填写个人简介。
✅ 正确：提供“建议的简介”功能，帮助他们在几秒钟内完成。
```

---

### 多赫蒂门槛 (Doherty Threshold)

**原则：** 当系统对用户操作的响应时间低于 400ms 时，生产力会大幅提升。

**应用：**

- **反馈:** 点击后立即给予视觉反馈。
- **加载:** 使用骨架屏 (Skeleton Screen)。
- **乐观 UI:** 在服务器响应前就先更新前端状态。
- **动效:** 使用微动效来掩盖轻微的延迟。
- **缓存:** 在后台预加载下一个页面或资产。
- **优先级:** 在重的高清晰度图像之前加载文本内容。

**示例：**

```
❌ 错误：按钮点击后 2 秒内没有任何反应。
✅ 正确：按钮立即改变颜色并显示“加载中”转圈。

❌ 错误：请求数据时显示空白白屏。
✅ 正确：使用骨架屏显示内容即将出现的灰色轮廓。
```

---

### 波斯塔尔定律 (鲁棒性原则 / Postel’s Law)

**原则：** 发送时要严谨，接收时要宽容。

**应用：**

- **错误处理:** 不要因为漏掉一个空格或连字符就报错。
- **格式化:** 同时接受 DD/MM/YYYY 或 MM/DD/YYYY 的日期格式。
- **输入:** 自动修剪开头/结尾的空白字符。
- **回退:** 如果用户没有上传头像，使用默认头像。
- **搜索:** 接受拼写错误并提供“你是不是想找...？”建议。
- **无障碍:** 确保网站在所有浏览器和设备上都能运行。

**示例：**

```
❌ 错误：因为用户在电话号码中输入了空格就直接拒绝。
✅ 正确：接受输入并自动去除空格。

❌ 错误：强制用户输入“January”而不是“01”或“Jan”。
✅ 正确：日期字段能够理解所有三种格式。
```

---

### 奥卡姆剃刀原则 (Occam’s Razor)

**原则：** 在同样可以解释现象的情况下，假设最少的那个解释通常是最好的。最简单的方案通常是最好的。

**应用：**

- **逻辑:** 减少不必要的点击。
- **视觉:** 只使用严格必要的字体/颜色。
- **功能:** 如果一个字段可以完成两个字段的工作，就合并它们。
- **文案:** 使用最短的文字来传达含义。
- **布局:** 移除不服务于目标的装饰性元素。
- **流程:** 除非绝对需要，否则避免分支路径。

**示例：**

```
❌ 错误：一个“登录”按钮先打开新页面，然后输入邮箱，再输入密码。
✅ 正确：在一个屏幕上同时询问两项内容的单个登录弹窗。

❌ 错误：在一个卡片上使用 5 种不同的字号和 4 种颜色。
✅ 正确：使用 2 种字号和 1 种强调色。
```

## 2. 视觉感知 (格式塔原则 / Gestalt Principles)

### 邻近原则 (Law of Proximity)

**原则：** 空间上相互靠近的物体倾向于被视为一组。

**应用：**

- **分组:** 将标签物理上靠近输入框。
- **间距:** 不相关的內容区块之间留出更大的边距。
- **卡片:** 卡片内的文字应比边框更靠近其关联的图片。
- **页脚:** 将法律链接聚在一起，远离社交媒体链接。
- **导航:** 将“用户”设置与“应用”设置分开分组。
- **表单:** 将地址字段分组在一起，与信用卡字段分开。

**示例：**

```
❌ 错误：表单中每行文本之间都有巨大且相等的间隙。
✅ 正确：标签与其输入框之间间距紧密，而各组之间间隙较大。

❌ 错误：“提交”按钮漂浮在页面中间，远离表单。
✅ 正确：“提交”按钮直接放在最后一个输入字段下方。
```

---

### 相似原则 (Law of Similarity)

**原则：** 视觉上相似的元素倾向于被视为一个整体。

**应用：**

- **一致性:** 所有可点击链接使用一致的颜色。
- **图标:** 一组图标应具有相同的线条粗细。
- **按钮:** 重要性相同的按钮具有相同的形状/尺寸。
- **排版:** 所有章节标题使用相同的 H2 样式。
- **反馈:** 所有“删除”操作应使用相同的颜色 (如红色)。
- **状态:** Hover 和 Active 状态在整个应用中必须一致。

**示例：**

```
❌ 错误：有的链接是蓝色，有的是绿色，有的只是加粗的黑色。
✅ 正确：应用中每个可点击的文本元素都是同一种蓝色。

❌ 错误：使用“蓝色按钮”执行“提交”，同时用同样的“蓝色按钮”执行“取消”。
✅ 正确：“提交”是实心蓝；“取消”是蓝色边框 (幽灵按钮)。
```

---

### 共同区域原则 (Law of Common Region)

**原则：** 在明确边界内的元素倾向于被视为一组。

**应用：**

- **容器化:** 使用卡片将图片和标题分组。
- **边框:** 使用线条将侧边栏与主信息流分开。
- **背景:** 为页脚使用不同的背景颜色。
- **弹窗:** 使用不同的盒子将弹窗与页面分离。
- **列表:** 使用交替背景色 (斑马纹) 区分行。
- **头部:** 横跨顶部的实色条将导航项分组。

**示例：**

```
❌ 错误：新闻列表中的文字和图片与其他故事重叠。
✅ 正确：每篇文章都包含在浅灰色背景上的白色卡片中。

❌ 错误：页脚与主体具有相同的背景颜色。
✅ 正确：深色背景的页脚清晰地将法律链接与页面内容分开。
```

---

### 连续性原则 (Law of Uniform Connectedness)

**原则：** 视觉上连接的元素 (如通过线条、箭头) 被视为比没有连接的元素更相关。

**应用：**

- **流程:** 使用线条连接进度向导中的步骤。
- **菜单:** 下拉菜单“触碰”或连接到它们的父按钮。
- **图表:** 在图表中连接数据点的线条。
- **关系:** 将切换开关连接到它控制的文本。
- **层级:** 文件目录的树状结构。
- **表单:** 将“信用卡”单选按钮连接到下方的字段集。

**示例：**

```
❌ 错误：一个 3 步设置，数字“1”、“2”、“3”散乱分布。
✅ 正确：用一条水平线连接“1”、“2”、“3”以显示顺序。

❌ 错误：漂浮的下拉菜单，没有接触开启它们的按钮。
✅ 正确：下拉菜单在视觉上“附着”在父按钮上。
```

---

### 简约原则 (简约法则 / Law of Prägnanz)

**原则：** 用户会倾向于将复杂的图像解释为最简单的形式。

**应用：**

- **清晰度:** 导航中使用清晰的几何图标。
- **减少:** 去掉不必要的 3D 纹理或阴影。
- **形状:** 倾向于标准长方形/圆形，而不是复杂的多边形。
- **焦点:** 为主要动作使用高对比度的剪影。
- **Logo:** 在小尺寸下也能识别的简单品牌标记。
- **UX:** 每个页面一个主要目标，保持“心理形状”简单。

**示例：**

```
❌ 错误：使用超写实的 3D 文件夹作为“文件”图标。
✅ 正确：使用文件夹的简单 2D 轮廓。

❌ 错误：使用多色、复杂的 Logo 作为加载转圈。
✅ 正确：使用简单的单色圆环。
```

---

### 图底关系原则 (Law of Figure/Ground)

**原则：** 人眼会自动区分物体 (图) 与其周围区域 (底)。

**应用：**

- **焦点:** 弹窗使用遮罩层 (Scrim) 来凸显内容。
- **深度:** 使用投影来暗示“图”位于“底”之上。
- **对比:** 深色底上的浅色文字 (反之亦然)。
- **模糊:** 背景模糊来突出前景文字。
- **导航:** 停留在页面內容上方的浮动粘性头部。
- **悬停:** 悬停时稍微抬高卡片，将其定义为“图”。

**示例：**

```
❌ 错误：一个弹窗没有任何阴影或边框，融入了页面。
✅ 正确：一个带有投影和变暗背景遮罩的弹窗。

❌ 错误：白色文本直接放在繁忙、多色的照片上。
✅ 正确：白色文本放在深色半透明的“遮罩”上。
```

---

### 焦点原则 (Law of Focal Point)

**原则：** 视觉上最显眼的部分首先捕获注意力。

**应用：**

- **入口:** 在焦点位置放置核心价值主张。
- **颜色:** 在中性 UI 中使用一种高亮度的“动作颜色”。
- **动效:** 在 CTA 上使用微小的动画来吸引眼球。
- **尺寸:** 最重要的统计数据应使用最大的字体。
- **排版:** 为标题使用粗重字体，为正文使用标准字体。
- **方向:** 使用箭头或视线 (人们看向按钮的图片)。

**示例：**

```
❌ 错误：主页有 5 个大小和颜色都相同的按钮。
✅ 正确：一个明亮颜色的巨大“开始使用”按钮。

❌ 错误：仪表盘上的“总收入”与“系统版本”大小相同。
✅ 正确：“总收入”以巨大的粗体数字显示在顶部中心。
```

## 3. 认知偏见与行为 (Cognitive Biases & Behavior)

### 蔡格尼克效应 (Zeigarnik Effect)

**原则：** 人们对未完成或中断的任务比已完成的任务记得更清楚。

**应用：**

- **游戏化:** 使用“个人资料完成度 60%”进度条。
- **参与感:** 在学习路径中预告下一个模块。
- **留存:** 显示尚未探索功能的“待办事项”列表。
- **反馈:** 未读消息的持久角标。
- **动力:** 完成一个步骤后立即显示“下一步”。
- **购物:** 购物车中的“完成订单”提醒。

**示例：**

```
❌ 错误：静默的引导过程，没有指示还剩什么。
✅ 正确：显示“5 个步骤已完成 3 个”的检查清单。

❌ 错误：在线学习应用即使视频只看了一半也显示勾选。
✅ 正确：进度环保持半满，直到视频看完。
```

---

### 目标梯度效应 (Goal Gradient Effect)

**原则：** 离目标越近，完成目标的动机越强。

**应用：**

- **动力:** 给予用户“虚假进展”(如 2 个免费印章)。
- **进展:** 将 10 个字段的表单拆成两个各 5 个字段的步骤。
- **反馈:** 在任务过半时庆祝里程碑。
- **动机 (Motivation):** 向用户展示他们距离 reward/status 有多近。
- **导航:** 使用面包屑显示距离终点还有多近。
- **加载:** 加载动画接近 100% 时加速。

**示例：**

```
❌ 错误：进度条从 0% 开始，感觉像是一场长跑。
✅ 正确：进度条从 20% 开始，因为用户通过打开应用“开始”了任务。

❌ 错误：结账流程中，“最终检查”感觉像是一个意外的第 5 步。
✅ 正确：清晰地标注步骤：“物流 > 支付 > 即将完成！”
```

---

### 峰终定律 (Peak-End Rule)

**原则：** 人们通过峰值时刻 (最强烈处) 和结尾时刻来评价体验。

**应用：**

- **成功:** 让“订单已确认”屏幕令人难忘。
- **愉悦:** 在产生价值的时刻添加纸屑或独特的动画。
- **支持:** 确保与聊天机器人的最后一次互动是有帮助的。
- **注销:** 即使当用户离开时，也让最后的退出过程整洁。
- **引导:** 以明确的“胜利”结束第一次会话。
- **错误处理:** 将 404 页面转化为有趣且有用的互动。

**示例：**

```
❌ 错误：经过 20 分钟的报税过程，应用只说“已提交”。
✅ 正确：“恭喜你！”页面，带有退税金额的摘要。

❌ 错误：游戏以简单的纯文本“Game Over”结束。
✅ 正确：摘要页面显示伴有庆祝音乐的高分。
```

---

### 审美响应效应 (Aesthetic-Usability Effect)

**原则：** 用户往往认为美观的设计更易用。

**应用：**

- **信任:** 高颜值的视觉可以为小的 Bug 赢得用户的“信任宽容”。
- **品牌:** 一致的高质量图像建立专业感。
- **参与感:** 漂亮的界面能让用户探索更久。
- **耐心:** 如果 UI 漂亮，用户对加载时间的容忍度更高。
- **信心:** 整洁的设计让复杂的工具感觉更易管理。
- **忠诚度:** 人们会与漂亮的产品建立情感纽带。

**示例：**

```
❌ 错误：银行应用排版错乱、颜色冲突。
✅ 正确：整洁、现代、带有平滑动画的银行应用。

❌ 错误：使用低分辨率、像素化的素材照片。
✅ 正确：使用高清、定制的品牌插图。
```

---

### 锚定偏差 (Anchoring Bias)

**原则：** 用户过度依赖第一眼看到的信息作为后续决策的基准。

**应用：**

- **定价:** 展示被划掉的原价。
- **层级:** 将最贵的“企业”方案放在最左侧。
- **排序:** 高亮“最受欢迎”作为第一项建议。
- **折扣:** 在显示最终价格前先声明“节省 20%”。
- **限制:** “每位客户限购 12 件”锚定了高价值的想法。
- **默认值:** 从较高的“建议捐赠”金额开始。

**示例：**

```
❌ 错误：只显示价格“$49”。
✅ 正确：显示“~~$99~~ $49 (5 折)”。

❌ 错误：按价格从低到高排列笔记本电脑列表。
✅ 正确：先显示高端“Pro”型号，让其他型号显得便宜。
```

---

### 社交证明 (Social Proof)

**原则：** 观察他人的行为来确定自己的行为。

**应用：**

- **验证:** 显示“加入另外 50,000+ 人”。
- **评论:** 星级评分和经过验证的客户证言。
- **Logo:** “值得信赖”部分显示合作伙伴品牌。
- **通知:** “Sarah 就在 5 分钟前购买了此商品”。
- **活动:** “目前有 300 人正在浏览此商品”。
- **证书:** 行业奖项和安全徽章。

**示例：**

```
❌ 错误：注册页面只有一个表单。
✅ 正确：注册页面写着“加入 200 万设计师”。

❌ 错误：匿名且没有姓名或照片的评论。
✅ 正确：评论包含真实面孔、姓名和“已验证买家”标签。
```

---

### 稀缺性原则 (Scarcity Principle)

**原则：** 人们对稀缺物体的估值更高。

**应用：**

- **紧迫感:** “库存仅剩 2 件”。
- **时间:** 促销活动的倒计时计时器。
- **访问:** “受邀制”内测或独占级别。
- **季节性:** “夏季限量版”产品。
- **低库存:** “即将补货 - 立即预订”。
- **需求:** “需求量大 - 有 10 人在购物车中添加了此商品”。

**示例：**

```
❌ 错误：永不结束且没有倒计时的促销。
✅ 正确：带有计时器的“每日特惠”。

❌ 错误：显示产品可用但没有库存数量。
✅ 正确：“此价格仅剩 3 件！”
```

---

### 权威偏差 (Authority Bias)

**原则：** 倾向于对权威人物的意见给予更高的准确性评价。

**应用：**

- **专业:** 使用“专家验证”或专业头像。
- **证书:** 信任印章 (Norton, ISO, HIPAA)。
- **媒体:** “TechCrunch/Forbes 报道”Logo。
- **认可:** 行业领袖或网红的评价。
- **语言:** 自信、专业且准确的文案。
- **品牌历程:** “成立于 1950 年”暗示长久和信任。

**示例：**

```
❌ 错误：健康博客由“管理员”撰写。
✅ 正确：健康文章“由心脏病专家 Jane Smith 博士审核”。

❌ 错误：安全应用没有任何认证提及。
✅ 正确：显示“ISO 27001 认证”和“Norton 安全”Logo。
```

---

### 损失规避 (Loss Aversion)

**原则：** 避免损失的痛苦远大于获得收益的快乐。

**应用：**

- **文案:** “不要丢掉你的折扣”。
- **试用:** “你的免费试用即将结束 - 立即保留你的数据”。
- **稀缺性:** “一旦错过，就永远错过了”。
- **购物车:** “不要错过你购物车中的商品”。
- **忠诚度:** “你已获得 500 积分 - 不要让它们过期”。
- **风险:** “30 天退款保证”(减少金钱“损失”感)。

**示例：**

```
❌ 错误：“点击这里获取 $10 优惠券”。
✅ 正确：“你有一笔 $10 余额待领取。在它今晚过期前使用它！”

❌ 错误：“取消你的订阅”。
✅ 正确：“如果你取消，你将失去对 50 个已存项目的访问权限”。
```

---

### 错误共识效应 (False-Consensus Effect)

**原则：** 我们倾向于高估他人与我们意见一致的程度。

**应用：**

- **测试:** 你不是用户 - 使用真实的受众进行测试。
- **调研:** 使用定性数据 (访谈) 和定量数据 (分析)。
- **偏差:** 使用“盲测评审”来避免个人偏好。
- **画像:** 坚持使用已建立的用户画像，而不是个人的直觉。
- **差异:** 测试不同背景/能力的用户。
- **客观性:** 使用热力图观察实际的用户行为。

**示例：**

```
❌ 错误：设计师在不测试的情况下决定一个功能是“直观的”。
✅ 正确：运行 A/B 测试来观察用户更喜欢哪个版本。

❌ 错误：因为“大家都懂英语”而完全使用英语构建应用。
✅ 正确：基于实际用户的地理位置数据添加本地化支持。
```

---

### 知识诅咒 (Curse of Knowledge)

**原则：** 一个领域内的专家倾向于假设非专家具备与他们同样水平的基础知识。

**应用：**

- **文案:** 避免术语，使用平实的语言。
- **引导:** 假设用户一无所知的教程。
- **工具提示:** 悬停时解释复杂术语。
- **结构:** 渐进式披露 (隐藏高级设置)。
- **标签:** 为导航使用图标 + 文本标签 (不要只依赖图标)。
- **支持:** 为新手提供全面的常见问题解答。

**示例：**

```
❌ 错误：错误消息显示“Exception: Null Pointer at 0x0045”。
✅ 正确：错误消息显示“发生错误。请尝试刷新”。

❌ 错误：使用“S3 Bucket Instances”等术语在云应用中导航。
✅ 正确：使用“文件存储”等简单术语。
```

---

### 进阶效应 (门面效应 / Foot-in-the-Door)

**原则：** 用户如果从一个小任务开始，就会致力于完成后续的大任务。

**应用：**

- **漏斗:** 在索要信用卡信息前先索要电子邮箱。
- **参与感:** 在注册前询问一个偏好 (如“深色模式？”)。
- **引导:** 使用一系列“快速是/否”的问题。
- **信任:** 在要求订阅前提供免费的 PDF/工具。
- **资料:** 先请用户上传照片，然后再填写简介。
- **销售:** 在主服务前提供低价的“导火索”产品。

**示例：**

```
❌ 错误：“开始免费试用”按钮立即要求信用卡信息。
✅ 正确：先索要邮箱和密码，然后再提供试用。

❌ 错误：在一页上显示所有 50 个调查问题。
✅ 正确：调查以一个简单的“是/否”问题开始。
```

## 6. 认知负荷管理 (Cognitive Load Management)

### 三种类型的认知负荷

| 类型                  | 定义               | 设计师的角色       |
| --------------------- | ------------------ | ------------------ |
| **原生 (Intrinsic)**  | 任务固有的复杂度   | 拆分成更小的步骤   |
| **外来 (Extraneous)** | 来自拙劣设计的负担 | **彻底消除！**     |
| **关联 (Germane)**    | 学习付出的努力     | 支持并鼓励这种努力 |

### 减少策略

**1. 简化 (减少外来负荷)**

```css
/* 视觉噪音 → 整洁 */
.card-busy {
    border: 2px solid red;
    background: linear-gradient(...);
    box-shadow: 0 0 20px...;
    /* 过载！ */
}

.card-clean {
    background: white;
    border-radius: 16px;
    box-shadow: 0 10px 30px -10px rgba(0, 0, 0, 0.1);
    /* 冷静、专注 */
}
```

**2. 渐进式披露**

```html
<!-- 隐藏复杂度直到需要 -->
<div class="filters">
    <div class="filters-basic">
        <!-- 常用过滤器可见 -->
    </div>
    <button onclick="toggleAdvanced()">高级选项 ▼</button>
    <div class="filters-advanced" hidden>
        <!-- 隐藏复杂的过滤器 -->
    </div>
</div>
```

## 8. 用户画像速查 (User Persona Quick Reference)

### Z 世代 (1997-2012 出生)

- **特征:** 数字原住民, 移动优先, 追求真实感, 注意力较短。
- **设计方法:** 鲜艳、激进的配色, 大面积排版, 垂直滚动, 快速游戏化交互。

### 千禧一代 (1981-1996 出生)

- **特征:** 重视体验, 购买前会研究, 具有社会价值观, 对价格敏感。
- **设计方法:** 莫兰迪色色调, 干净的无衬线字体, 卡片式布局, 平滑有目的的动效。

### X 世代 (1965-1980 出生)

- **特征:** 独立、自力更生, 重视效率, 对营销持有怀疑态度。
- **设计方法:** 专业、值得信赖的颜色, 传统的排版, 清晰的层级结构。

## 10. 心理学检查清单 (Psychology Checklist)

### 发布前最终检查：

- [ ] **席克定律:** 选项是否通过渐进披露减少了决策疲劳？
- [ ] **菲茨定律:** 移动端最重要按钮是否易于触达？
- [ ] **多赫蒂门槛:** 交互响应是否在 400ms 内？是否有骨架屏？
- [ ] **雅各布定律:** 布局是否遵循了主流 web 惯例？
- [ ] **孤立效应:** 主 CTA 是否全局最显眼？
- [ ] **峰终定律:** “成功”页面是否制造了愉悦感？
- [ ] **审美响应:** UI 精度是否足以建立用户的初始信任？
- [ ] **信任信号:** 安全徽章、社交证明、专家认证是否可见？
- [ ] **损失规避:** 文案是否强调了用户“保留”而非仅仅“获得”？
- [ ] **无障碍:** 对比度、键盘导航、Prägnanz (简约图标) 是否达标？

---

> **记住**: 心理学是思考，不是复制。每一个项目都值得基于其独特的上下文和用户进行独立思考。**拒绝现代 SaaS 安全港 (Avoid the Modern SaaS Safe Harbor)!**
