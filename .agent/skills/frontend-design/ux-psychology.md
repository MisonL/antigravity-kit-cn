# UX 心理学参考 (UX Psychology Reference)

> 深入 UX 定律、情感化设计、信任构建与行为心理学。

---

## 1. 核心 UX 定律 (Core UX Laws)

### 希克定律 (Hick's Law)

**原理：** 做决策的时间随着选择数量的增加呈对数增长。

```
决策时间 = a + b × log₂(n + 1)
其中 n = 选择数量
```

**应用：**

- 导航：顶级菜单限制在 5-7 个
- 表单：分步骤 (渐进式披露)
- 选项：尽可能提供默认选择
- 筛选：优先展示常用项，隐藏高级项

### 菲茨定律 (Fitts' Law)

**原理：** 移动到目标的时间取决于目标的距离和大小。

```
时间 = a + b × log₂(Distance / Size + 1)
```

**应用：**

- CTA (行为号召按钮)：做得够大
- 触摸目标：移动端至少 44x44pt
- 无限边缘：屏幕边缘是最容易击中的目标 (适用于菜单)
- 距离：相关操作放在一起 (如：输入框 + 提交按钮)

### 雅各布定律 (Jakob's Law)

**原理：** 用户大部分时间花在该别的网站上。这意味着他们希望你的网站像他们已经认识的其他网站一样工作。

**应用：**

- 遵循标准模式 (如：Logo 回首页，汉堡菜单在移动端)
- 不要为了不同而不同
- 使用熟悉的术语和图标

### 米勒定律 (Miller's Law)

**原理：** 普通人只能在其工作记忆中保持 7 (±2) 个项目。

**应用：**

- 分块 (Chunking)：用空格分隔电话号码
- 分组：将相关选项放在一起
- 不要让用户同时记住超过 5 件事

### 帕累托原理 (80/20 法则)

**原理：** 80% 的结果来自 20% 的原因。

**应用：**

- 关注 20% 的核心功能 (MVP)
- 优化 20% 用户最常使用的路径
- 修复造成 80% 错误的 20% 的 Bug

## 2. 认知偏差 (Cognitive Biases)

### 锚定效应 (Anchoring)

**原理：** 人们过于依赖他们收到的第一条信息 ("锚")。

**应用：**

- 定价：先展示高价方案，让中等价格看起来便宜
- 折扣：展示原价 (锚) 和 现价

### 稀缺性偏差 (Scarcity)

**原理：** 人们更加重视稀缺的东西。

**应用：**

- 限量版
- "仅剩 2 个库存"
- 限时优惠

### 社会证明 (Social Proof)

**原理：** 在不确定的情况下，人们会模仿他人的行为。

**应用：**

- 评价和评分
- "10,000+ 用户信任"
- "其他人也买了"
- Logo 墙 (权威证明)

### 峰终定律 (Peak-End Rule)

**原理：** 人们主要根据体验的高峰 (最好的或最坏的) 和结束时的感受来评判一段体验，而不是每一刻的总和。

**应用：**

- 庆祝成功时刻 (高峰)
- 确保流程结束时有愉快的确认 (终)
- 哪怕是错误信息也要写得体贴 (避免低谷)

### 冯·雷斯托夫效应 (Von Restorff Effect)

**原理：** 多个相似物体中，那个不同的最容易被记住。(隔离效应)

**应用：**

- 主要 CTA 颜色必须不同于次要按钮
- 定价表中推荐的方案要做突出处理

### 蔡格尼克效应 (Zeigarnik Effect)

**原理：** 人们对未完成或中断的任务比完成的任务记得更清楚。

**应用：**

- 进度条 (显示还没做完)
- 任务列表 (划掉已完成的)
- 游戏化中的"每日任务"

## 3. 格式塔原理 (Gestalt Principles)

### 邻近性 (Proximity)

**原理：** 彼此靠近的物体被视为一组。

**应用：**

- 标题和其段落的距离 要小于 和上一段的距离
- 表单标签要靠近其输入框
- 卡片内的元素间距要小于卡片间的间距

### 相似性 (Similarity)

**原理：** 看起来相似的元素被视为具有相同的功能。

**应用：**

- 链接颜色必须一致
- 主按钮风格必须一致
- 卡片样式复用

### 连续性 (Continuity)

**原理：** 眼睛会沿着路径、线条或曲线移动。

**应用：**

- 使用线条引导视线指向 CTA
- 轮播图露出下一个项目的一部分，暗示可滚动
- 对齐文本以创建垂直阅读线

### 闭合性 (Closure)

**原理：** 大脑会自动填充缺失的部分以形成完整的图像。

**应用：**

- 图标简化 (不需要画出每一条线)
- 加载指示器 (圆圈不需要闭合)
- 露出屏幕边缘的内容暗示更多内容

## 4. 信任心理学 (Trust Psychology)

### 信任金字塔

1.  **基线信任 (Baseline):** 网站看起来专业吗？没有拼写错误？加载快吗？
2.  **功能信任 (Functional):** 它能工作吗？有 Bug 吗？
3.  **价值信任 (Value):** 它对我有用吗？
4.  **情感信任 (Emotional):** 我感觉好吗？这种连接是否真实？

### 信任信号设计

```css
/* 信任徽章容器 */
.trust-badge-container {
    display: flex;
    align-items: center;
    gap: 8px;
    padding: 12px 16px;
    background: #f0fdf4; /* 浅绿 = 安全 */
    border-radius: 2px; /* 锐利 = 精准感 */
    font-size: 14px;
    color: #166534;
}

/* 安全表单指示 */
.secure-form::before {
    content: "🔒 安全表单";
    display: block;
    font-size: 12px;
    color: #166534;
    margin-bottom: 8px;
}

/* 证言卡片 */
.testimonial {
    display: flex;
    gap: 16px;
    padding: 24px;
    background: white;
    border-left: 4px solid var(--primary); /* 坚实的支撑线 */
}
```

### 降低焦虑的设计

- **清晰的文案:** 避免行话
- **无隐性成本:** 尽早显示总价
- **易于撤销:** "随时取消"
- **确认反馈:** "您的订单已保存"

## 5. 情感化设计 (Emotional Design)

### 诺曼的三个层次

1.  **本能层 (Visceral):** 第一印象。"它看起来好看吗？"
    - _应用:_ 美学、色彩、高质量图片。
2.  **行为层 (Behavioral):** 可用性。"它好用吗？"
    - _应用:_ 性能、易用性、预期反馈。
3.  **反思层 (Reflective):** 故事与意义。"它代表了什么？"
    - _应用:_ 品牌故事、个人身份认同、记忆点。

### 创建愉悦 (Delight)

- **微交互:** 点赞时的心跳动画
- **文案:** 友好、幽默的错误信息 (视上下文而定)
- **插画:** 空状态下的个性化展示
- **个性化:** "早上好，Mison"

## 6. 行为设计模型 (Fogg Behavior Model)

**行为 (B) = 动机 (M) × 能力 (A) × 提示 (P)**

### 要发生行为，必须同时具备：

1.  **动机 (Motivation):** 用户想要做。
    - _设计:_ 强调价值、紧迫感、社会认同。
2.  **能力 (Ability):** 用户容易做。
    - _设计:_ 简化流程、减少步骤、默认选项 (Hick's Law)。
3.  **提示 (Prompt):** 提醒用户做。
    - _设计:_ 清晰的 CTA、通知、红点。

### 应用策略

- 如果动机低，不要强推 (增加价值先)。
- 如果能力低，简化任务 (Make it easy)。
- 如果在错误的时间提示，用户会烦。

## 7. 色彩心理学 (Color Psychology)

> 参见 [color-system.md](color-system.md) 获取详细指南。

简要回顾：

- **红:** 激情、危险、错误 (高唤醒)
- **蓝:** 信任、冷静、稳定 (低唤醒)
- **黄:** 快乐、警告、注意
- **绿:** 自然、生长、成功、金钱
- **橙:** 友好、便宜、活力
- **紫:** 皇室、神秘、创意 (AI 常用，需克制)
- **黑:** 奢华、力量、现代
- **白:** 纯洁、干净、极简

## 8. 排版心理学 (Typography Psychology)

### 字体个性

- **Serif (衬线):** 传统、可信、正式、文学。
    - _适用:_ 金融、新闻、时尚品牌。
- **Sans-serif (无衬线):** 现代、干净、直接、极简。
    - _适用:_ 科技、初创、SaaS。
- **Display (展示/手写):** 友好、独特、创意。
    - _适用:_ 儿童产品、个人博客。

### 易读性与信任

- 难读的字体会降低任务完成率，也会降低信任度 (Hygiene 因素)。
- 大字号、高对比度通常被认为更自信。

## 9. 道德设计 (Ethical Design)

### 避免暗黑模式 (Dark Patterns)

- **不要** 诱导点击 (Bait and switch)。
- **不要** 隐藏取消订阅按钮 (Roach motel)。
- **不要** 强迫连续性 (自动续费陷阱)。
- **不要** 羞辱用户 ("不，我不想要省钱")。

### 以人为本

- 尊重用户的时间。
- 尊重用户的隐私。
- 设计用于赋能，而不是成瘾。

## 10. UX 检查清单

在设计评审时：

- [ ] **减少了选择吗？** (Hick's Law)
- [ ] **点击目标够大吗？** (Fitts' Law)
- [ ] **遵循了惯例吗？** (Jakob's Law)
- [ ] **最重要的东西最突出吗？** (Von Restorff)
- [ ] **分组逻辑清晰吗？** (Gestalt)
- [ ] **建立了信任信号吗？**
- [ ] **避免了暗黑模式吗？**
- [ ] **考虑了峰终体验吗？**

---

> **记住**: 心理学是工具，不是操控手段。用它来帮助用户更轻松地达成**他们的**目标，而不是利用他们的弱点。
