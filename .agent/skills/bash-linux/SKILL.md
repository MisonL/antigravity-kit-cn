# Bash & Linux 技能

## 安全原则

1.  **慎用 `rm -rf`**: 始终检查当前路径。
2.  **权限最小化**: 避免使用 `sudo` 运行 npm/pip。
3.  **管道检查**: `curl | bash` 之前先检查脚本内容。

## 常用命令速查

| 任务         | 命令                                 |
| :----------- | :----------------------------------- | --------------------------- |
| **查找文件** | `find . -name "*.ts"` 或 `fd "*.ts"` |
| **内容搜索** | `grep -r "TODO" .` 或 `rg "TODO"`    |
| **进程管理** | `ps aux                              | grep node`, `kill -9 <PID>` |
| **端口占用** | `lsof -i :3000`                      |
| **磁盘空间** | `du -sh *`                           |

## Shell 脚本最佳实践

始终在脚本开头加上：

```bash
#!/bin/bash
set -e  # 遇到错误立即退出
set -u  # 使用未定义变量报错
set -o pipefail # 管道中任一命令失败则整体失败
```
