# 结构化头脑风暴

在写代码之前，使用 `/brainstorm` 探索多种方案。

## 概览

`/brainstorm` 会启用**结构化构思模式**。
它不会直接跳到实现，而是至少提供 3 种方案，分析利弊，辅助你做出选择。

<FeatureGrid>
    <Feature title="不写代码" description="只讨论思路" />
    <Feature title="3+ 方案" description="对比选择" />
    <Feature title="利弊分析" description="客观权衡" />
    <Feature title="你来决定" description="最终拍板" />
</FeatureGrid>

## 流程

<StepList>
  <Step number={1} title="描述挑战">
    告诉智能体你要探索什么，并说明约束条件。

    ```bash
    /brainstorm 我的 SaaS 应用的用户认证系统
    ```

  </Step>
  <Step number={2} title="查看方案">
    智能体给出 3+ 方案，包含优缺点与工作量评估。
  </Step>
  <Step number={3} title="做出决策">
    选定方案后，用 `/enhance` 或 `/create` 进入实现。
  </Step>
</StepList>
